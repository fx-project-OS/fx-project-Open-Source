Personalstammblatt eines internen Mitarbeiters


Eine übersichtliche Anordnung sämtlicher relevanten Daten wie Mitarbeiterdaten, Vertragsdaten, Bankdaten, Adressdaten oder Kommunikationsdaten zu einer Person.

Dabei kann die Fülle der Informationen bequem durch Anhaken der entsprechenden Checkboxen erhöht oder verringert, sowie der Zeitraum der anzuzeigenden Sonderdaten wie Zeitdaten oder Abwesenheitsdaten frei festgelegt werden.




Anzeigekriterien für das Personenstammblatt

In der Maske Anzeigekriterien für das Personenstammblatt wird die Auswahl getroffen welche Informationen zu einer Person aufgelistet werden sollen. Voraussetzung ist hierbei die Beschränkung auf eine bestimmte Person, sowie auf einen bestimmten Zeitraum. Die Mitarbeiterdaten werden immer angezeigt, egal welche Auswahlkriterien getroffen werden.

Der anzuzeigende Zeitraum muss über die beiden Eingabefelder Zeitspanne von und Zeitspanne bis angegeben werden. Wird keine Eingabe getätigt wird der aktuelle Kalendermonat voreingestellt.

Durch Auswahl der Felder Zeitdaten, Urlaub, Krankheit oder Schulung werden die entsprechenden Daten zur ausgewählten Person ermittelt und aufsummiert angezeigt. Dabei bedeutet [Summe] Anzeige der Zwischensumme pro Jahr und [Detailliert] pro Kalendermonat.

Durch Anhaken der Checkbox-Felder Vertragsdaten, Bankdaten, Adressen, Kommunikationsdaten, Fähigkeiten oder Personenhierarchie werden die entsprechenden Daten zur ausgewählten Person ermittelt und in dem gleichlautenden Übersichtsblock angezeigt.

Die gewünschten Daten werden nach einem Klick auf den Button Anzeigen ermittelt und in der Maske Personalstammblatt aufbereitet angezeigt, wo sie über den [Datei-Manager] weiterverarbeitet werden können.



Feldbeschreibungen

Person:

In diesem Auswahlfeld wird die Person gewählt, deren Daten je nach Aktion bearbeitet oder angezeigt werden soll.

Zeitspanne von:

In diesem Feld wird angezeigt, von welchem Zeitpunkt an Daten für ein Projekt aufgelistet werden sollen.

Im Eingabemodus kann hier ein Datum entsprechend den Benutzereinstellungen eingegeben werden.

Zeitspanne bis:

In diesem Feld wird angezeigt, bis zu welchem Zeitpunkt die Daten für ein Projekt aufgelistet werden sollen.

Im Eingabemodus kann hier ein Datum entsprechend den Benutzereinstellungen eingegeben werden.

Zeitdaten:

Im Auswahlfeld Zeitdaten kann angegeben werden, ob und wie die Arbeitsstunden der Person in dem ausgewählten Zeitraum angezeigt werden sollen. [Summe] bedeutet je Kalenderjahr, [Detailliert] je Kalendermonat.

Urlaub:

Im Auswahlfeld Urlaub kann angegeben werden, ob und wie die genommenen Urlaubstage der Person in dem ausgewählten Zeitraum angezeigt werden sollen. [Summe] bedeutet je Kalenderjahr, [Detailliert] je Kalendermonat.

Krankheit:

Im Auswahlfeld Krankheit kann angegeben werden, ob und wie die Krankheitstage der Person, in denen sie gefehlt hat, in dem ausgewählten Zeitraum angezeigt werden sollen. [Summe] bedeutet je Kalenderjahr, [Detailliert] je Kalendermonat.

Schulung:

Im Auswahlfeld Schulung kann angegeben werden, ob und wie die Tage der Person, in denen Sie auf Schulung war, in dem ausgewählten Zeitraum angezeigt werden sollen. [Summe] bedeutet je Kalenderjahr, [Detailliert] je Kalendermonat.

Auswahlmöglichkeit aller angelegten Proj.-Kategoriegruppen

Vertragsdaten:

Durch Anhaken des Feldes Vertragsdaten kann bestimmt werden, ob die entsprechenden Daten im gleichlautenden Block aufgelistet werden sollen oder nicht.

Bankdaten:

Durch Anhaken des Feldes Bankdaten kann bestimmt werden, ob die entsprechenden Daten im gleichlautenden Block aufgelistet werden sollen oder nicht.

Adressen:

Durch Anhaken des Feldes Adressen kann bestimmt werden, ob die entsprechenden Daten im gleichlautenden Block aufgelistet werden sollen oder nicht.

Kommunikationsdaten:

Durch Anhaken des Feldes Kommunikationsdaten kann bestimmt werden, ob die entsprechenden Daten im gleichlautenden Block aufgelistet werden sollen oder nicht.

Fähigkeiten:

Durch Anhaken des Feldes Fähigkeiten kann bestimmt werden, ob die entsprechenden Daten im gleichlautenden Block aufgelistet werden sollen oder nicht.

Personenhierarchie:

Durch Anhaken des Feldes Personenhierarchie kann bestimmt werden, ob die entsprechenden Daten im gleichlautenden Block aufgelistet werden sollen oder nicht.
Personenhierarchie bedeutet hier sowohl Vorgesetzte als auch Untergeordnete.

ZE n.verrech.:

Hier kann angegeben werden, ob die getätigte Zeiterfassung bei der Rechnungsstellung fakturiert werden soll oder nicht.
