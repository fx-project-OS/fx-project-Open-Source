Administration > Berechtigungen > Rechteübersicht

Übersicht der vergebenen Zugriffsrechte


Alle Zugriffsrechte von Projekten, Dokumenten und Reports einer einzelnen Person können angezeigt werden.




Benutzer und Referenz auswählen

Benutzer und Referenz auswählen

Referenz und Benutzer auswählen zu denen die vorhandenen Rechte angezeigt werden.

Bei Projekten wird Projektbaum angezeigt mit berechtigten Projekten und Pfaden, die zu Berechtigten Projekten führen.

Auf jeden Fall werden nur Projekte aus Projektmenü angezeigt (nur die Projekte, die der angemeldete Benutzer sehen darf und es gelten dieselben Kriterien, die im Projektmenü ausgewählt sind).

Die Projekte, auf die der ausgewählte Benutzer Rechte hat, werden mit dunklerem Hintergrund betont.



Feldbeschreibungen

Benutzer:

Dieses Feld beinhaltet einen internen Schlüssel für jeden Benutzereintrag (Benutzer-ID).

Referenz:

Auswahl der Referenz, für welche dem Benutzer die Rechte zugewiesen wurden.


Übernehmen

Durch einen Klick auf diesen Button werden die eingegebenen Daten gespeichert und die Programmausführung fortgesetzt.




Benutzer hat explizit zugewiesene Rechte für folgende Referenzen

Es wird eine Liste mit Projekten, Reporten, Dokumenten usw. angezeigt, für die der Benutzer eine Berechtigung hat

Es wird Liste angezeigt, für alle ausgewählten Benutzer mit Berechtigung.

Diese Liste beinhaltet: Projekte, Reporte, Dokumente usw. (abhängig von der ausgewählten Referenz).

Die Zugriffsrechte für Dokumente sind zu einzelnen Aktionen zugeteilt.



Feldbeschreibungen

Bezeichnung:

Bezeichnung des Datentyps.

Anz.:

Dieses Feld gibt an, ob das bezügliche Feld oder die Programmfunktion für dieses Profil angezeigt werden darf.

Anl.:

Dieses Feld gibt an, ob das bezügliche Feld für dieses Profil angelegt werden darf.

Änd.:

Hier wird angegeben ob das Feld für dieses Profil geändert werden darf.

Lös.:

Dieses Feld gibt an, ob das bezügliche Feld für dieses Profil gelöscht werden darf.
