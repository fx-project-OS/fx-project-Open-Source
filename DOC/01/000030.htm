Administration > Profile/Benutzer > Benutzerdefinition

Erstellen von Benutzern und Zuordnen zu einer bereits angelegten Person


Der Zugang zur Software fx-project erfolgt über einen Benutzer-Zugang, welcher über ein Passwort geschützt ist. Dem Benutzer muss eine gültige E-Mail-Adresse zugeordnet werden, damit dieser für generierte Mails erreichbar ist. Bei der Definition des Benutzers kann außerdem ein Benutzerprofil sofort zugeordnet werden. Des weiteren ist es möglich, die Datenanzeige bezüglich der Kosten festzulegen. Soll der Benutzer administrative Zugriffsrechte erhalten, ist der entsprechende Haken in dem zugeordneten Feld zu setzen. Ein Benutzer wird immer zu einer definierten Person (Mitarbeiter (intern / extern) / Auftraggeber etc.) zugeordnet werden.




Benutzerauswahl

Auswahl des zu bearbeitenden Benutzers.

In der Benutzerauswahl wählt man den gewünschten Benutzer zur weiteren Verarbeitung der Daten aus.

Diese können im Benutzerblock je nach Aktion angezeigt, angelegt, geändert oder gelöscht werden.

Der Hauptadministrator kann nicht gelöscht werden und wird daher in diesem Modus nicht angezeigt.

HINWEIS:
Die Einträge für das Feld "Benutzername" können unter dem Menüpunkt "Administration/Benutzerverwaltung/Benutzer definieren" verwaltet werden.
Falls Sie auf eines dieser Programme kein Recht haben bzw. es nicht in Ihrem Hauptmenü erscheint, dann wenden Sie sich bitte an Ihren Administrator, Projektleiter oder Vorgesetzen.



Feldbeschreibungen

Benutzer:

In Benutzer können Benutzer gewählt werden.


Auswählen

Durch einen Klick auf diesen Button werden die ausgewählten Daten weitergeleitet.




Benutzerblock

Ändern der Daten des gewählten Benutzers.

Im Benutzerblock bearbeitet man die Daten des gewählten Benutzers.

Um die Daten eines anderen Benutzers zu ändern, wählt man ihn in der Benutzerauswahl aus.

Der Hauptadministrator kann nicht gelöscht werden und wird daher in diesem Modus nicht angezeigt.

Über die Administrator-Prg. gibt man dem Benutzer Administrator-Rechte.

Neues Passwort und Passwort wiederholen müssen übereinstimmen. Wird ein Passwort eingegeben wird dieses abgeändert und der Benutzer aktiviert, falls dieser inaktiv oder gesperrt war. Wird kein Passwort eingegeben wird der Benutzerstatus nicht verändert.

An die E-Mail-Adresse des Benutzers werden alle vom Programm ausgehenden E-Mails versendet. Daher sollte sie korrekt und aktuell sein.

HINWEIS:
Die Einträge für das Feld "zugehörige Person" können unter dem Menüpunkt "Personen/(entspr. Personentyp)" verwaltet werden.
Falls Sie auf eines dieser Programme kein Recht haben bzw. es nicht in Ihrem Hauptmenü erscheint, dann wenden Sie sich bitte an Ihren Administrator, Projektleiter oder Vorgesetzen.



Feldbeschreibungen

Benutzername:


In Benutzername wird der Account einer Person gespeichert. Über diesen Namen, das Passwort und die Mandanten ID loggt man sich in das System ein. Die Namenskonvention ist dabei frei gewählt bzw. von der Firma frei bestimmt.

Benutzer-ID:

Diese Benutzer-ID dient der eindeutigen Kennzeichnung eines Benutzers innerhalb eines Mandanten.

E-Mail-Adresse:

An die hier eingegebene E-Mail-Adresse werden sämtliche Nachrichten des Systems für den Benutzer geschickt. Daher sollte sie immer korrekt und aktuell sein.

Gültig von:

Datum, ab wann ein Datensatz gültig ist.

Login:

Das Neues Passwort enthält das persönliche Login-Kennwort eines Benutzers. Zur eigenen Sicherheit sollten keine einfachen Passwörter verwendet werden.
Sollte das Passwort vergessen werden, kann der Eigentümer beim Administrator ein neues anfordern, das er später wieder ändern kann.

Administr.-Prg.:

Administrator-Prg. gibt an, ob der Benutzer über Administrator-Rechte verfügt und dessen Programme ebenso nutzen darf.

Gültig bis:

Datum, bis wann ein Datensatz gültig ist.

Passwort wdh.:

Um Tippfehler auszuschließen bzw. vorzubeugen muss das neue Passwort ein weiteres Mal in das Feld Passwort wiederholen eingegeben werden.

zugeh. Person:

Wählen sie in zugehörige Person die zugehörige Person des Benutzers aus um auf die Daten (Projekte, ...) der Person zugreifen zu können.

Sprache:

Die eingestellte Sprache.

Kosten:

In Kosten werden dem Benutzer Rechte auf Kostenfelder gegeben bzw. entzogen. Dies ist von Vorteil, wenn sie den Kunden Anzeigerechte auf Projektdaten geben, dieser aber keine Kosten sehen darf.

Ben.Profil Grp.:

In diesem Feld kann man über ein Auswahlmenü die gewünschte Benutzerprofilgruppe selektieren.


Multi-Funktion

Durch Klick auf diesen Button werden die Änderungen durchgeführt.
