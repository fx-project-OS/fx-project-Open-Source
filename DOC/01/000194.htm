Referenzauswahl

Referenz für Rechtevergabe

Hier muss zuerst eine Referenzkategorie (Projekte, Dokumente oder Reports) und dann eine Unterkategorie zu dieser Referenz ausgewählt werden.

Wenn "Rechte- Vergabe" aus einer anderen Programmfunktion abgerufen wird, sind diese Werte schon vorbelegt und werden nur angezeigt.

Zugriffsrechte verwalten

Zugriffsrechte verwalten

In dieser Maske können die Zugriffsrechte an bestimmte Abteilungen, Benutzergruppen oder einzelner Benutzer vergeben werden.
Wenn der Benutzer schon ein direktes Recht auf ausgewählte Referenzen (durch seine Zuordnung zu diesen oder übergeordneten Projekt) wird dies mit blauem Hintergrund angezeigt

Die Zugriffsrechte für Dokumente sind Aktionen zugeteilt und bedeuten im Einzelnen:


HINWEIS:
Alle: Alle Zugriffsrechte werden gewährt.

abrufen: Der Benutzer darf das Dokument downloaden.

ein-/auschecken: Der Benutzer darf das Dokument zur
Bearbeitung down- bzw. uploaden. (In der Zwischenzeit darf die Datei dann nur noch runtergeladen werden.

archivieren: Die Datei wird archiviert und taucht in künftigen Suchabfragen nicht mehr auf.

löschen: Die Datei wird gelöscht.



Feldbeschreibungen

Nur Kundendaten:

Aktivieren bzw. Deaktivieren eines Eintrages durch Setzen eines Häkchen in die "Checkbox"

Bezeichnung:

Bezeichnung der Referenz.

Alle:

Durch das Drücken von Alle auswählen werden alle Checkboxen mit einem Haken vorselektiert. Diese Änderungen müssen mit einem weiteren Button wie z.B. OK bestätigt werden.

Anz.:

Dieses Feld gibt an, ob das bezügliche Feld oder die Programmfunktion für dieses Profil angezeigt werden darf.

Anl.:

Dieses Feld gibt an, ob das bezügliche Feld für dieses Profil angelegt werden darf.

Änd.:

Hier wird angegeben ob das Feld für dieses Profil geändert werden darf.

Lös.:

Dieses Feld gibt an, ob das bezügliche Feld für dieses Profil gelöscht werden darf.
