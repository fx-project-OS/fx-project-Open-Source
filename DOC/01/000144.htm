Import von Projekten einer MS-Project® CSV-Datei in fx-project®


Liest eine mit MS-Project® erzeugte CSV-Datei ein und erstellt daraus eine neue Projektstruktur in fx-project. Dadurch können Daten sehr einfach zwischen fx-project und MS-Project® ausgetauscht werden.


HINWEIS:
Die verwendeten Felder der CSV-Datei können frei definiert und angepasst werden.




Projekte aus MS-Project® importieren

Hier können Sie bestehende Projekte aus MS-Project® nach fx-project importieren.

Das Projekt in MS-Project® muss als Textdatei (mit Tabulator getrennt) gespeichert werden.

Folgende Felder sind notwendig (links Name in MS-Project®, in Klammern der Name, unter dem das Feld exportiert werden muss, um von fx-project erkannt zu werden):
- Gliederungsnummer (Position)
- Name (Name/Projektbezeichnung)
- Anfang (Anfang/Soll_Beginn)
- Ende (Ende/Soll_Ende)
- Arbeit (Arbeit/Aufwand_Soll)
- Vorgänger (Vorgänger/abhängig_von)

Optionale Felder:
- Erstellt (Erstellt/Erstelldatum)
- Notizen (Notizen/Beschreibung)
- Priorität (Priorität)
- %Abgeschlossen (Prozent_Abgeschlossen/Fertigstellungsgrad => setzt dann Projektstatus auf geplant oder aktiv)

Wählen Sie eine gültige (Text-)Datei und füllen Sie die folgenden Auswahlfelder mit den jeweils von Ihnen gewünschten Personen.

Geben Sie des Weiteren an, in welcher Einheit die Arbeit aus MS-Project® exportiert wurde (Minuten, Stunden o.ä.) und in welchem Format das Datum vorliegt.

HINWEIS:
Keine der zu importierenden Datumsangaben darf Textzeichen enthalten.



Feldbeschreibungen

GIF-Bild:

Gibt an welches Icon bei diesem Eintrag verwendet wird.

Dokument:

Dokumentendatei

Auswahl des Vorgangs (Projekt, Teilprojekt, Aufgabe) für den man berechtigt ist.:

Im Feld Projekt werden die Vorgänge (Projekte oder Aufgaben) aufgelistet, für die man die entsprechenden Rechte besitzt und die entsprechen Filterkriterien gelten.

Anspr. Auftrag.:

Dieses Feld zeigt den Ansprechpartner bzw. den Kunden. Durch Selektion eines Ansprechpartners wird in einigen Fällen (z.B. Projektdefinition) automatisch auch ein Kunde gewählt, da jeder Ansprechpartner genau einem Kunden zugeordnet ist.

Angelegt wird ein Ansprechpartner über die Eingabemaske im Menüpunkt "Stammdaten".

Projektleiter:

In diesem Feld wird der Leiter eines Projektes angezeigt bzw. muss aus einer Auswahlliste gewählt werden. Der Projektleiter wird einmal im Mutterprojekt festgelegt, und wird dann als VORSCHLAG in alle darunter liegenden Projekten vorausgewählt.

Grün angezeigte Personen im Auswahlfeld sind Projektleiter, welche bereits mindestens einem Projekt in dieser Rolle zugeordnet sind.

Angelegt werden Projektleiter über die Rollen-Maske mit Rolle "Projektleiter".

Projektmanager:

Gibt an, welche Person der Projektmanager dieses Projektes sein soll.

Dieses Feld dient lediglich als weiteres Entscheidungskriterium in div. Reports, der Projektmanager genießt keinerlei Zusatzrechte o.ä., diese liegen ausschließlich beim Projektleiter

Der Projektmanager wird nur in Projekten gesetzt und wird dann in den jeweiligen Aufgaben des Projektes ebenfalls selektiert.

Grün angezeigte Personen im Auswahlfeld sind Projektmanager, welche bereits mindestens einem Projekt in dieser Rolle zugeordnet sind.

Angelegt werden Projektmanager über die Rollen-Maske mit Rolle "Projektmanager".

Stellvertr. PL:

Hier kann ein stellvertretender Projektleiter für ein Projekt angegeben werden, der den Projektleiter vertritt, falls dieser abwesend ist.

Grün angezeigte Personen im Auswahlfeld sind stellvertretende Projektleiter, welche bereits mindestens einem Projekt in dieser Rolle zugeordnet sind.

Angelegt werden stellvertretende Projektleiter über die Rollen-Maske mit Rolle "Projektleiter".

Datum:

Dient zur Eingabe oder Anzeige eines Tagesdatums. Bei einer erforderlichen Datumseingabe ist meistens der Standardwert (das heutige Datum) bereits vorbelegt.

Trennzeichen:

Trennzeichen für die Exportfelder bei Projektexport in MS_Projekt.

Datumsformat:

Legt das Format fest, in dem alle Datumsangaben angezeigt und eingelesen werden. Das Datum muss mit folgenden Variablen definiert werden. Trennzeichen beliebig:

dd: Tag (day) 2-stellig
mm: Monat 2-stellig
yy: Jahr (Year) 2-stellig
yyyy: Jahr 4-stellig

Datum Bottom-Up:

Datum nach der Bottom-Up-Methode anpassen

Das Datum nach der Bottom-Up-Methode anpassen, d.h. alle weiter oben liegenden Ebenen werden auf das nachfolgende Datum gestreckt, falls eine Zeitüberlappung stattfindet.

1509:

Textfeld

Variables Textfeld, welches programmgesteuert angepasst werden kann.

Prefix einfügen:

Prefix (dt. Präfix oder auch Vorsilbe genannt) bezeichnet ein dem Wortstamm vorangestellte Zeichenfolge, hier um eine Gruppe von zusammengehörigen Wörtern zu erzeugen.

Wird z.B. beim Projektnamen verwendet um eine einheitliche Projektzugehörigkeit bereits beim Namen zu erreichen. Der hierbei zu verwendende Platzhalteraufbau kann in den Administrator-Einstellungen definiert werden.

Feldbeschreibungen

Zeile:

Gibt an in welcher Zeile der Maske dieses Feld erscheinen soll.

Projekt-Nr.:

Durchlaufende eindeutige Nummerierung.

Projekt:

In diesem Feld wird der Projektname des aktuell ausgewählten Projektes angezeigt.

Gep. Start:

Hier wird das Soll-Startdatum des Projektes angezeigt bzw. angegeben, d.h. ab wann laut Planung mit dem Projekt zu beginnen ist.

Gep. Ende:

Hier wird das Soll-Enddatum des jeweiligen Bezugsobjekts, z.B. Projekt, Zeitdaten, Reisedaten etc. angezeigt bzw. angegeben, d.h. bis wann laut Planung das Objekt abgeschlossen sein soll.

Priorität:

Gibt die Dringlichkeit des (Teil-)Projektes an.
Je höher die Priorität eines Projektes ist, desto höher wird dieser im Kontrollzentrum eines Mitarbeiters angezeigt.

Aufw. Soll:

Der Soll-Aufwand entspricht dem geplanten Zeitaufwand in Stunden, Manntagen oder Mannjahren etc. Der geschätzte Aufwand wird bei der Definition von Aufgaben eingetragen. Bei Projekten/Teilprojekten entspricht dieser Wert der Summe aller untergeordneten Aufgaben.

Zeitformat:

Mit dieser Benutzereinstellung kann ein Benutzer festlegen, in welchem Format er Zeiten angezeigt bekommen möchte. Das Format ist gleichzeitig das Eingabe-Format für diesen Benutzer.
hh:mm (Stunden, Minuten jeweils zweistellig)
hh:mm:ss (Stunden, Minuten, Sekunden jeweils zweistellig)
hh.mm

Beschreibung:

Erläuternde Beschreibungen, Detailangaben, Bemerkungen oder ähnliches.
