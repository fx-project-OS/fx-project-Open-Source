Administration > Datenbank > Datenbanksicherung

Sichern oder Wiederherstellen der eigenen kompletten Mandantendaten


Die Datenbanksicherung dient zum Sichern (= Backup) und Wiederherstellen (= Restore) der mandanteneigenen Daten. Eine Rücksicherung kann nur auf dieselbe Version erfolgen, die zum Zeitpunkt der Sicherung aktiv war. Lizenzierungsinformationen sind davon ausgenommen. Ferner können veraltete Datenbanksicherungen gelöscht, sowie aktuelle im Browser angezeigt bzw. runtergeladen werden.




Datenbanksicherungen

Auflistung aller Datenbanksicherungen des aktuellen Mandanten.

Je nach ausgewählter Aktion kann die Datenbank gesichert, wiederhergestellt oder eine Sicherungsdatei gelöscht werden.



Feldbeschreibungen

Nur Kundendaten:

Aktivieren bzw. Deaktivieren eines Eintrages durch Setzen eines Häkchen in die "Checkbox"

Tab.-Ans.:

Ist die Checkbox aktiviert, dann werden sämtliche zur grafischen Aufbereitung notwendigen Daten auch in einem Report aufgelistet, welcher als PDF inkl. Grafik angezeigt und runtergeladen werden kann.

Version:

Die Versionsnummer wird beim normalen Speichern eines geänderten Dokumentes um 0,001 erhöht, bei ausdrücklich erwünschtem Versionswechsel oder bei Überlauf des Fließkommabereichs (bei 999 Änderungen ohne Versionswechsel) um 1.

Manuell gewechselte Versionen werden als neue Datei gespeichert und aktiviert, die alte Version wird als inaktiv markiert.

Dateiname:

In dieses Feld muss der Name, der erzeugten Datei eingegeben werden.

Datum:

Dient zur Eingabe oder Anzeige eines Tagesdatums. Bei einer erforderlichen Datumseingabe ist meistens der Standardwert (das heutige Datum) bereits vorbelegt.

Anzahl:

Anzahl der Einträge

Größe:

Dieses Feld gibt die physikalische Größe der Datei an. Mit dieser Information kann man beispielsweise abschätzen, wie lange ein Download erfahrungsgemäß bei der gewohnten Bandbreite (-> Internetzugang) dauern würde.

Link:

Dieses Feld ist optional. Ist ein Link eingetragen, so wird die automatische Benutzerrecht-Abfrage übergangen und das Feld Trans_ID gelöscht.


Speichern

Button zum speichern von Daten.

Löschen

Nachdem der Button Löschen gedrückt wurde, werden die gewählten Datensätze gelöscht. Sollte der Datensatz doch noch gebraucht werden, erhalten Sie eine Fehlermeldung.
