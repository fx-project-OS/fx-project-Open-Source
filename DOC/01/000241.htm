Definition von Kostenträger, Kostenstelle, Kostenart und Leistungsträger für Kostenrechnung


Verwalten (Neuanlage, Ändern oder Löschen) von allen Kosten-Stammdaten in folgenden vier Kostenstammkategorien:

  • Kostenträger
  • Kostenstelle
  • Kostenart
  • Leistungsart

Jeder dieser Kategorien muss eine eindeutige Kostennummer zugewiesen werden, deren Format frei für den ganzen Mandanten festgelegt werden kann. Ferner kann optional jeder Kategorie eine "Bezeichnung", "Art" oder "Beschreibung" hinterlegt werden, sowie ein evtl. Gültigkeitszeitraum.




Auswahl der Kosten-Stammdatenkategorie

Auswahl, zu welcher Stammdatenkategorie und in welcher Sprache Kostenstellen erstellt werden sollen.

Kosten-Stammdatendefinition

Neuanlage, Ändern oder Löschen von Stammdatendefinitionen.

Zu jeder Kosten-Stammdatenkategorie können hier beliebig viele Stammdatendefinitionen festgelegt werden mit einzigartigen Kostennummern, dazugehörigen beliebigen Bezeichnungen sowie optionalem Beschreibungstext.
Ebenso kann hier bereits festgelegt werden zu welcher Kostenstellenart die Stammdatendefinition gehören soll.

Wird eine Leistungsart bearbeitet kann hier ferner definiert werden ob die Leistungsart [produktiv] oder [nicht produktiv] ist.

Des weiteren können auch Gültigkeitsdauer für die einzelnen Stammdatendefinitionen festgelegt werden, sodass bei einer Neuzuordnung immer nur der aktuellste Pool verwendet wird.

Kosten-Nr.:

Eindeutige Identifizierungsnummer von Kostenstammdaten wie Kostenträger, Kostenstellen, Kostenarten oder Leistungsarten.

Projekt:

Ist ein Projekt abhängig von einem anderen, dann kann das abhängige Projekt erst nach Abschluss des Vorgängers begonnen werden.

Kommt es zu Verspätungen beim Vorgänger, bedeutet dies, dass sich auch der Nachfolger (abhängiges Projekt) zeitlich nach hinten verschiebt, dies wird aber nicht automatisch durch das Programm bewerkstelligt, sondern muss vom User explizit ausgeführt werden!

Jedes Projekt kann von jedem, bereits angelegten Projekt, in Abhängigkeit gesetzt werden.

In diesem Feld wird intern festgelegt, bei welchem Mandanten dieser Datensatz abzurechnen ist.

Somit kann z.B. ein Mitarbeiter unter seinem normalen Mandanten eine Zeiterfassung tätigen, jedoch abgerechnet wird diese bei einem anderen Mandanten.
