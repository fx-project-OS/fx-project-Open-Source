Creation > Projects > Order Requests

Creation of planned requirements that become a main project after release


Manage (create, modify or delete) orders, i.e. planned requirements. The title, a contact person, completion date, order cause and priority as well as order target and note can be entered as pure text values.

A new main project can be created from each order with a single mouse click.




Determine selection criteria

Select order directly or limit list by criteria.



Field Descriptions

Order title:

The order title contains the name or designation of the planned operation (project). When an order is activated, a new master project is created from this title.

Client of a project.:

Here you can select the client of the activity (project, subproject or task).

Filter categories for projects:

Define filter categories for projects or select them in a selection field.

Year:

The year is specified in this field.


Display

By clicking on this button, the list is generated and displayed. Depending on the defined selection criteria, the list is filtered accordingly.




Order list

Display of all orders which correspond to the set criteria.

By clicking on the [ACTIVATE] icon , an order can be created as a parent project. At the same time, all project managers will be notified by e-mail about the new project. By clicking on the [ACTIVATE] icon the information about the order can be viewed again.



Field Descriptions

Order Title:

The order title contains the name or designation of the planned operation (project). When an order is activated, a new master project is created from this title.

Proj. No.:

This is the directory number for a project.

The directory number cannot be changed for parent projects.

The directory number is automatically adjusted when deleting or moving.

By manually changing the directory number, you can move projects within a hierarchy.

Contact Prs. C.:

This field shows the contact person or the customer. By selecting a contact person, in some cases (e.g. project definition) a customer is automatically selected as well, since each contact person is assigned to exactly one customer.

A contact person is created via the input mask in the menu item "Master data".

Completion:

Latest planned date by which the order or the associated project is to be completed.

Link:

This field is optional. If a link is entered, the automatic user rights query is bypassed and the Trans_ID field is cleared.




Data sheet

Convenient management and creation of orders.

Based on the selected order, a new parent project is automatically generated and all project managers are informed about it by e-mail if the project checkbox is ticked.

If an order is deleted, associated parent projects remain unaffected. However, if the parent project has not yet been subdivided into other projects, it will be deleted as well.

Project:

By setting the checkbox a project will be generated from the order.

Order Cause:

As the name suggests, this field contains the cause, the reason for an order. This field is a lookup value and can be freely extended there.

Order Priority:

As the name suggests, this field contains the priority of an order. This field is a lookup value and can be freely extended there.

Order Goal:

Detailed description of the objective of the contract.

Annotation:

This field displays general comments about a project.
