Field Descriptions

Client Folder:

Folder of the client

The client folder is the subfolder on the file system in the fx-project directory with the number of the current client.

The client folder contains special settings files of the client, such as configuration files, person graphs, chat logs or statistics data, etc.

Configuration Type:

Type of configuration file

The configuration type indicates the type of configuration file, i.e. is it a configuration file for emails, invoices etc.?

Configuration File:

Name of the configuration file

Configuration file name defined by fx-project, i.e. [cfg_email.cfg] for emails, [cfg_plnw.cfg] for statement of work reports, [cfg_ppst.cfg] for graph reports, [cfg_prng.cfg] for invoices, and [cfg_pstd.cfg] for all other PDF reports.


Edit

Button for editing the data record.




Edit configuration file

Manage configuration files, i.e. create, edit or delete them.

The configuration file shows the path and name of the current configuration file, which is currently being edited. If possible and allowed, an icon is offered on the right, with which the configuration file can be deleted if desired.

The toolbar consists of various support tools to upload graphics, convert units of measurement or display command templates:


  • Upload image
    Dient to upload (= upload) graphics to be able to include them in the configuration file. Depending on the configuration file type, only a certain number of graphics can be uploaded at a time. For the e-mail configuration file, the limit is max. 4 pieces, for the PDF configuration files at max. 8 pieces at the same time.
    Only [.gif], [.jpg] or [.png] graphics are allowed as graphic format.


  • Include image
    Dient to include the uploaded graphics in the configuration files. A list of all existing graphics is displayed. With a click on a graphic, the corresponding command is written in the status bar, which can then be copied into the configuration file at the desired location using [cut & paste].
    The displayed command is aligned for a right-justified alignment of the graphic in the PDF document, so this is excellent for logos, etc.


  • Calculator
    Dient for converting PDF length units. This makes it very easy to convert millimeter (mm) specifications to the point (pt) units used with PDF documents. All you have to do is enter the new mm value in the corresponding field. If the field is left, the value is converted automatically into the point unit.
    One mm corresponds thereby to 2.83501683501 PDF points. Or vice versa: One PDF point corresponds to 0.35273159145 mm.


  • image
    Writes the command-.template [^Ppic|x|y|sf|align|width|height] in the status bar to embed a graphic into the PDF-document. This command can then be copied into the configuration file at desired location using [cut & paste] and customized.
    Parameter description: (The pipe symbol (= |, vertical line) acts as a separator of the individual parameters)
    • ^P stands as a shortcut for the instruction to include an image (P for Picture).
    • pic stands as a placeholder for the file name of the graphic to be displayed. The graphic must be located in the corresponding client folder - which is automatically the case if the graphic was uploaded with the corresponding icon.
    • x X-coordinate (= value on the right value axis, i.e. value from left to right on the screen) of the upper left corner of the graphic, if left-aligned; center, if centered; upper right corner, if right-aligned.
    • y Y-coordinate (= value on the superscript axis, i.e. value from top to bottom on the screen) of the upper left corner of the graphic, if left-aligned; midpoint, if centered; upper right corner, if right-aligned.
    • sf Scale factor from 0.00 (= 0%) to 2.00 (= 200%) of the original size. Default value = 1.00, i.e. do not display graphic reduced or enlarged. (This parameter is optional)
    • align Alignment of the graphic, i.e. left (= left-aligned), center (= centered) or right (= right-aligned). Default value = left, i.e. left-justified. (This parameter is optional)
    • width Width of the graphic in pixels. If this value is specified, the width of the graphic will be stretched or compressed according to the ratio of the specified width to the original width. (This parameter is optional)
    • height Height of the graphic in pixels. If this value is specified, the graphic height will be stretched or compressed according to the ratio of the specified height to the original height. (This parameter is optional)


  • text
    Writes the command template [^Ttext|x|y|sf|align|mode|lw] in the status bar to write a text into the PDF-document. This command can then be copied into the configuration file at desired location using [cut & paste] and customized.
    Parameter description: (The pipe symbol (= |, vertical line) acts as a separator of the individual parameters)
    • ^T stands as a shorthand for the instruction to write a text.
    • text stands as a placeholder for the text string to be displayed. The pipe must not be used, as this acts as a separator of the individual parameters.
    • x X-coordinate (= value on the right-value axis, i.e. value from left to right on the screen) of the left start point of the text baseline, if left-aligned; midpoint, if centered; right end point, if right-aligned.
    • y Y coordinate (= value on the superscript axis, i.e. value from top to bottom on the screen) of the left start point of the text baseline, if left-aligned; midpoint, if centered; right end point, if right-aligned.
    • sf Font size in PDF points. Default value = 8.00 points. (This parameter is optional)
    • align Alignment of the text, i.e. left (= left-aligned), center (= centered), or right (= right-aligned). Default value = left, i.e. left-justified. (This parameter is optional)
    • mode font mode, ie. normal (= without effects), bold (= boldface), outline (= outlined), shadow (= shaded), or stamp (= reverse shaded). Default value = normal, i.e. without effects. (This parameter is optional)
    • lw Stroke width in PDF points e.g. for border effect. Default value = 0.25 points. (This parameter is optional)


  • Box
    Writes the command-.Template [^Bx1|y1|x2|y2|rgb|fb|wd|bd|lw] in the status bar to draw a box (= filled rectangle) in the PDF-document. This command can then be copied into the configuration file at desired location using [cut & paste] and adjusted.
    Parameter description: (The pipe (= |, vertical line) acts as a separator of the individual parameters)
    • ^B stands as a shorthand for the instruction to draw a box.
    • x1 X coordinate (= value on the right value axis, i.e. value from left to right on the screen) of the upper left corner of the rectangle to be filled.
    • y1 Y coordinate (= value on the high value axis, i.e.i.e. value from top to bottom on the screen) of the upper left corner of the rectangle to be filled.
    • x2 X coordinate (= value on the right value axis, i.e. value from left to right on the screen) of the lower right corner of the rectangle to be filled.
    • y2 Y coordinate (= value on the high value axis, i.e. value from top to bottom on the screen) of the lower right corner of the rectangle to be filled.
    • rgb Rectangle color in hexadecimal format for Rred - Ggreen - Bblue color components. Default value = FFFF (= white). (This parameter is optional)
    • fb Frame pattern consisting of 4 numbers (0 or 1) with 1 = draw frame or 0 = do not draw frame. The 4 numbers are to be entered in the following order: Frame Left, Top, Right, Bottom. Default value = 1111, i.e. draw complete frame. (This parameter is optional)
    • wd White portion of the background pattern, i.e. a number between 0.00 (= none) and 1.00 (= full). Default value = 0.00, i.e. without white component. (This parameter is optional)
    • bd Black portion of the background pattern, i.e. a number between 0.00 (= without) and 1.00 (= full). Default value = 0.00, i.e. without black component. (This parameter is optional)
    • lw Line width in PDF points of the rectangular frame. Default value = 0.35 points. (This parameter is optional)


  • Rectangle
    Writes the command template [^Rx1|y1|x2|y2|wd|bd|lw] in the status bar to draw a rectangle in the PDF-document. This command can then be copied into the configuration file at desired location using [cut & paste] and customized.
    Parameter description: (The pipe (= |, vertical line) acts as a separator of the individual parameters)
    • ^R stands as a shorthand for the instruction to draw a rectangle.
    • x1 X-coordinate (= value on the right value axis, i.e. value from left to right on the screen) of the upper left corner of the rectangle.
    • y1 Y-coordinate (= value on the high value axis, i.e.i.e. value from top to bottom on the screen) of the upper left corner of the rectangle.
    • x2 X coordinate (= value on the right value axis, i.e. value from left to right on the screen) of the lower right corner of the rectangle.
    • y2 Y-coordinate (= value on the high value axis, i.e. value from top to bottom on the screen) of the lower right corner of the rectangle.
    • wd White portion of the frame pattern, i.e. a number between 0.00 (= none) and 1.00 (= full). Default value = 0.00, i.e. without white portion. (This parameter is optional)
    • bd Black portion of the frame pattern, i.e. a number between 0.00 (= without) and 1.00 (= full). Default value = 0.00, i.e. without black portion. (This parameter is optional)
    • lw Line width in PDF points of the rectangular frame. Default value = 0.35 points. (This parameter is optional)


  • line
    Writes the command template [^Lx1|y1|x2|y2|wd|bd|lw|rgb] in the status bar to draw a line in the PDF-document. This command can then be copied into the configuration file at desired location using [cut & paste] and customized.
    Parameter description: (The pipe (= |, vertical line) acts as a separator of the individual parameters)
    • ^L stands as a shorthand for the instruction to draw a line.
    • x1 X-coordinate (= value on the right value axis, i.e. value from left to right on the screen) of the upper left corner of the line.
    • y1 Y-coordinate (= value on the high value axis, i.e. value from top to bottom on the screen) of the upper left corner of the line.
    • x2 X coordinate (= value on the right value axis, i.e. value from left to right on the screen) of the lower right corner of the line.
    • y2 Y coordinate (= value on the high value axis, i.e. value from top to bottom on the screen) of the bottom right corner of the line.
    • wd White portion of the line pattern, i.e. a number between 0.00 (= none) and 1.00 (= full). Default value = 0.00, i.e. without white portion. (This parameter is optional)
    • bd Black portion of the line pattern, i.e. a number between 0.00 (= without) and 1.00 (= full). Default value = 0.00, i.e. without black component. (This parameter is optional)
    • lw Line width in PDF points of the line. Default value = 0.35 points. (This parameter is optional)
    • rgb Line color in hexadecimal format for Rred - Ggreen - Bblue color components. Default value = 000000 (= black). (This parameter is optional)



The status bar displays the messages of the respective actions, or the command templates.

In the actual input window, the configuration file can be edited and customized according to your own requirements.

The Preview button opens a preview of the current configuration file in a new window. Pure sample data is used for display to simulate how the final result would look like.

The Save button saves the changes to the configuration file.


NOTE:
If a button is mistakenly clicked which would overwrite the changes already made to the current configuration file, this is indicated by a JavaScript hint. If confirmed positively, the changes will still be saved.
For a more detailed description incl. examples of the command templates please read the PDF document [_DOK/fx-project 05.2 Workflow - Example for PDF configuration file.pdf] available in the fx-project documentation folder.
