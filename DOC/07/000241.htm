Definition of cost object, cost center, cost type and service provider for cost accounting


Manage (create, modify or delete) all cost master data in the following four cost master categories:

  • Cost object
  • Cost center
  • Cost type
  • Activity type
.
Each of these categories must be assigned a unique cost number, the format of which can be freely defined for the entire client. Furthermore, each category can optionally be assigned a "designation", "type" or "description", as well as a possible validity period.




Selection of the cost master data category

Selection for which master data category and in which language cost centers are to be created.



Field Descriptions

Basic Cost Category:

Master data category of the cost centers, for example, cost object, cost element, etc.

Language:

Assignment of a language in table "Language

The target start date of the project is displayed or specified here, i.e. from when the project is to be started according to planning.

The target end date of the respective reference object, e.g. project, time data, trip data, etc. is displayed or specified here, i.e. by when the object should be completed according to planning.

Display

By clicking on this button, the list is generated and displayed. Depending on the defined selection criteria, the list is filtered accordingly.




Cost master data definition

Create, change or delete master data definitions.

For each cost master data category, any number of master data definitions can be defined here with unique cost numbers, associated arbitrary designations and optional descriptive text.
Also, it can already be defined here to which cost center type the master data definition should belong.

If an activity type is processed, it can also be defined here whether the activity type is [productive] or [non-productive].

Furthermore, the validity period for the individual master data definitions can be defined, so that only the most current pool is used in case of a reassignment.



Field Descriptions

1509:

Cost master category (cost object, cost center, cost element or activity type)

Date from which a record is valid.

Valid to:

Text:

Variable text field that can be customized programmatically.

Label:

Cost Center Type:

Possible assignment type of the individual cost master definitions, such as project, person, material, etc.

Productive:

Determines whether an activity type is productive (= value-added) or not.

Description:

Explanatory descriptions, details, comments or similar.


Save

By clicking on this button, the changes are made. The text of the button depends on the current action mode.




Cost master data list

List of master data for cost centers including additional information.



Field Descriptions

Only Cust. Data:

Activate or deactivate an entry by placing a tick in the "checkbox".

Cost No.:

Unique identification number of cost master data such as cost objects, cost centers, cost elements or activity types.

Project:

If one project is dependent on another, then the dependent project can only be started after the predecessor has been completed.

If there are delays in the predecessor, this means that the successor (dependent project) is also delayed, but this is not automatically managed by the program, but must be explicitly executed by the user!

Each project can be set from each, already created project, in dependence.

Cost Label:

Designation of a unique cost center.

Type:

Possible assignment type of the individual cost master definitions, such as project, person, material, etc.

Prod.:

Determines whether an activity type is productive (= value-added) or not.

AS-ID:

This field is used internally to determine the client for which this data record is to be settled.

Thus, for example, an employee can make a time recording under his normal client, but it will be accounted under another client.

Valid from:

Date from which a record is valid.

Valid to:

Date until when a record is valid.

Edit:

Icon for changing an entry. If you click this button, you will usually enter the change mode.

Delete:

Icon to delete an entry.


Delete

After pressing the button Delete the selected records will be deleted. If the record is still needed, you will receive an error message.
