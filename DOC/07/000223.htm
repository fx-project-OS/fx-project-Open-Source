Filter settings in fx-projectÂ® can be named and help to reduce the amount of data and simplify e.g. extensive evaluations


With the help of the data filter, the amount of data used can be reduced with regard to persons, time and projects. This restriction reduces the display and increases the clarity. For example, the number of projects can be reduced to two, so that the corresponding reports are only applied to these two projects.

The filter can be switched on or off at the top right (red or green rectangle to the right of the filter symbol). In the masks in which the filter can be used, a filter symbol is visible in the selection mask of the program function in the upper right corner.

Field Descriptions

Text Add. (Head):

Text field for general text input.

Title:

In this field the official title of a person can be selected, e.g. "Professor", "Doctor" etc.

Person:

The field Person_Completed is identical with exactly one field "Name". It stands for a person who has completed a task/problem.

This field normally has the same value as the field Person_responsible, unless the person who solved the problem would be different from the one who was initially responsible for the problem.

Name:

In this field the family name of the person or the name of the legal entity (e.g. company name) is displayed or entered.

Project Team:

Project Team:

Group of people assembled as a team for a specific project.

Superior:

Field containing the Person_ID of the superior of a person.

Department:

Indicates the department in which the employee works.

Person:

blank

Skill:

Selecting a specific skill or skill group to group employees.

Person Type:

Indicates what type of person it is.

Inactive:

By setting the checkbox, inactive persons (employees, clients, etc.) will also be displayed.

Date:

Used to enter or display a date of the day. If a date entry is required, the default value (today's date) is usually already preset.

Date Entry:

With the help of the field "Date specification" you can directly display a desired day.

This option is preferable if you want to "jump" over a longer period of time, because here only one date has to be entered instead of a tedious scrolling forward or backward.

Act. Start:

Indicates the start date of the respective reference object, e.g. projects, time data, travel data, etc.

Act. End:

The actual end date of the respective reference object, e.g. project, time data, travel data, etc. is displayed or specified here.

Show Project:

With the help of this field it can be selected how the projects of the employees should be displayed. If the field is selected, then each individual project of an employee is displayed in a separate line, otherwise all projects are displayed in one line, whereby in the case of a project overlap the name of the project with the higher start date is displayed. The employee workload is added up in both cases of project overlaps.

Project:

ALIAS field for project ID

Name of Project:

This field displays the project name of the currently selected project.

Projects From:

If this checkbox is activated, all projects from this start date will be displayed.

Projects Until:

If this checkbox is activated, all projects up to this date are displayed.

Main Project:

Top level of a project structure, under which all subprojects and tasks are subordinated.

Customer:

Here you can select the client of the activity (project, subproject or task).

Project Manag.:

Specifies which person should be the project manager of this project.

This field only serves as a further decision criterion in various reports, the Project Manager does not enjoy any additional rights or similar, these lie exclusively with the 1st Project Leader.

The Project Manager is only set in projects and is then also selected in the respective tasks of the project.

Green displayed persons in the selection field are project managers, which are already assigned to at least one project in this role.

Project managers are created using the role mask with role "Project Manager".

1st Project L.:

In this field the manager of a project is displayed or must be selected from a selection list. The project manager is defined once in the master project, and is then preselected as a PROPOSAL in all underlying projects.

Green displayed persons in the selection field are project leaders, which are already assigned to at least one project in this role.

Project managers are created using the role mask with role "Project Manager".

Prj.-Category:

Define filter categories for projects or select them in a selection field.

Status:

This field shows how far a project has already progressed. In input mode, you can assign an attribute from a selection list to a project here, which reflects whether or how far a project is already advanced.

The status can also be used to decide whether project data should be included in a report or not.

Priority:

Indicates the urgency of the (sub)project.
The higher the priority of a project, the higher it is displayed in the control panel of an employee.

Displ. Tasks:

Here you can define whether tasks (and possibly to-dos) should also be displayed in the project selection list or not. If this checkbox is not set, only projects and subprojects are displayed.

Hide Archive:

By setting this "archiving indicator", complete mother projects can be archived, i.e. the data is retained, but these projects are no longer displayed in the selection fields or in the project menu. This makes it possible to reduce the list of projects.
A mother project can only be "archived" if it has already been completed, i.e. the status of the mother project and consequently of all subordinate projects must be set to "completed"
.

Only Order Pr.:

If this field is activated in the filter settings, only projects generated from orders are displayed.
