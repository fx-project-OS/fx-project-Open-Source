Field Descriptions

Date:

With the help of the field "Date specification" you can directly display a desired day.

This option is preferable if you want to "jump" over a longer period of time, because here only one date has to be entered instead of a tedious scrolling forward or backward.

CW:

Number of the week in which the day of the current time recording record lies.

Higher-Level Proj.:

The number you enter in this field controls how many project levels are displayed in the report.

Sublevel "0" corresponds to the highest level.

Person:

This selection field is used to select the person whose data is to be edited or displayed depending on the action.

Project:

The Project field lists the tasks (projects or tasks) for which you have the appropriate rights and the corresponding filter criteria apply.

Disp. service.:

Determines whether maintenance tasks should be displayed or not.

A maintenance task is a task that does not cause any workload because, for example, the effort was planned with 0. Useful, for example, for recurring long-running tasks without exactly predictable scheduling.

Disp. Piled Up T.:

If set, stacked tasks, i.e. tasks from the past that have not yet been completed, are also displayed.

Only Weekly Plan:

If set, only weekly tasks that have already been set will be displayed, regardless of any other selection criteria that may have been set.
Weekly tasks are set in the change mode of the corresponding program function.


Display

By clicking on this button, the list is generated and displayed. Depending on the defined selection criteria, the list is filtered accordingly.

Today

Button of the date selection. After clicking you will find yourself at today's date.




Project weekly schedule


Field Descriptions

Time:

Used to enter or display a time.

Project No.:

This is the directory number for a project.

The directory number cannot be changed for parent projects.

The directory number is automatically adjusted when deleting or moving.

By manually changing the directory number, you can move projects within a hierarchy.

Path:

In this field the path to a file is specified, this can also be changed directly in the mask.

Reference:

A reference number may be entered in this field.

It is used for a possible reference creation to other programs.

Tasks:

Here you can define whether tasks (and possibly to-dos) should also be displayed in the project selection list or not. If this checkbox is not set, only projects and subprojects are displayed.

Project Location:

Location where the project will be carried out. For construction projects, this can also be the construction site.

Persons:

Here you can select which employees should be included in the cost report.

Material ID:

Internal key for the Material table.

Night:

Overnight stay
