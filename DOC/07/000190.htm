Reporting > Times > Time Cost Report

Listing of the billable time recording of employees in a certain timespan


Detailed listing of all time recording hour costs in a freely definable period per employee and task.

As a filter criterion, in addition to the "person" and the "task", it is also possible to restrict by means of the "period" or the "client". The calendar display type is also freely adjustable between "daily", "monthly" or "annual overview".




Set display criteria for the cost report.

Set display criteria for the cost report.

Define the selection criteria according to which a cost report of the individual projects is to be prepared for the corresponding customer. In particular, the date to be displayed ("Time period from", "Time period to") can be set, as well as the underlying project.

If you have the appropriate rights, then a selection list will appear in the "Person" field, in which you can specify an employee for whom you want to see the cost report.

For client 1 the whole thing even cross-client.

SELECTION OPTIONS:
Display type
  • Automatic
    		• The program independently selects the most appropriate display format for the entered time period.
  • Day(s), Month(s), Year(s)
    		• The entered time span will be split into the selected time unit.

    Note: This will not work if "All employees" is selected, as here the employee names are the sort order.
    Project/Task
  • All projects
    		• All projects and subprojects are listed, to which the employee is assigned, and are included in the calculation.
  • Without projects
    		• Like "All projects", but all projects/tasks are not displayed, but only flow into the calculation.
  • Project (selected)
    		• Only the selected project with all subprojects will be listed, and will be included in the calculation.
    Person
  • All employees
    		• All employees of a firm, no matter if within or across clients, are listed. However, only those are displayed, which also really have entries for the selected criteria.
  • Intra-client employees
    		• Like "All employees" , but only intra-client.
  • Cross-client employees
    		• Like "All employees" , but only cross-client.
  • No employees
    		• Like "All employees" , but these are not listed individually, but their costs are added up to the corresponding projects.
  • Employee (selected)
    		• Only the cost report for the selected employee will be generated.
    Travel expenses
  • Travel expenses. Show in own line
    		• All travel expenses written down for a project will only be shown in the following line.
  • Add travel cost. Add to hourly costs
    		• All travel costs written down for a project will be added to the normal EK/VK.
  • Show travel costs alone
    		• Only the incurred travel costs for a project will be listed.
  • Do not include travel costs
    		• Only the normal costs for a project are listed, all travel costs are not calculated.


    NOTE:
    Note about travel expenses: Even if a record was marked as "not allocable" during time recording and entries were made for travel expenses at the same time, then such records are basically always included in the calculation of travel expenses.



    Field Descriptions

    Time Span from:

    This field displays the point in time from which data for a project should be listed.

    In input mode, a date can be entered here according to the user settings.

    Customer:

    Here you can select the client of the activity (project, subproject or task).

    Time Span to:

    This field displays the date up to which the data for a project should be listed.

    In input mode, a date can be entered here according to the user settings.

    Person:

    This selection field is used to select the person whose data is to be edited or displayed depending on the action.

    Display Type:

    Specifies how certain data should be formatted within a report.

    Project:

    The Project field lists the tasks (projects or tasks) for which you have the appropriate rights and the corresponding filter criteria apply.

    Show Empty Entries:

    By checking or unchecking this field, you can determine whether irrelevant entries (=blank entries) are to be displayed or not.

    If, for example, a month is listed for a project, but only two days in this month were worked for this project, you can decide here whether the entire month or only these two days should be displayed.

    Disp. Absent.:

    Determination of whether absences, i.e. days with missing time recording, should be displayed in the past or not.

    Release:

    The release indicates whether the record has been released for settlement.

    Show Unknown:

    If this checkbox is checked, "unknown" tasks will also be displayed.

    Disp. Unassign.:

    Determine whether empty assignments, that is, days without assigned plan tasks, should be displayed in the future or not.

    Clearable:

    Specifies whether or not to display offsetting entries.

    Serv.-Type:

    The activity type in cost accounting is a reference value that indicates the qualitative technical description of the production activity. Here, it can be classified between productive (=value-adding) and non-productive types.

    Disp. Estimat.:

    Determine whether or not plan estimates, that is, days with assigned plan tasks, should be displayed in the future.

    Invoiced:

    Specifies whether or not to display billed entries.

    with max. invoiceable hours:

    If this field is activated, the estimated billable hours are determined from the maximum of TARGET and ACTUAL hours, i.e. in the normal case, a complete working day is used for the calculation of earnings.

    Sort by Travel:

    If this checkbox is ticked, the trips are sorted and grouped by trip type incl. trip start and end, and not by current date.

    Travel Costs:

    Here you can select how the travel expenses entered in the time recording are to be included in the listed calculation.

    Possible settings:


    Display travel expenses in separate row:
    - Travel expenses are included and displayed on the next line of the project to which they are assigned.

    Add travel expenses to employee costs:
    - Travel expenses are included and added to employee costs in the same row of the project to which they are assigned.

    Show Travel Expenses Alone:
    - Only all travel expenses incurred alone will be displayed in the listing, meaning that all other employee hourly costs will not be included in the calculation.

    Do not add travel expenses:
    - Travel costs are not included in the calculation, only employee hourly costs are listed.

    Day Specifications:

    Specify whether day specifications are displayed or not, that is, whether day specifications are to be added to the travel expenses or not.

    Vouchers:

    The payment type for travel expense receipts specifies who paid the receipt.

    There are two possibilities:

    • Company
      The receipt was paid by the company, e.g. usually for airline tickets, hotel accommodation etc.
    • Private
      The receipt was paid privately by the employee, e.g. usually for food, parking, taxi etc.


    Display

    By clicking on this button, the list is generated and displayed. Depending on the defined selection criteria, the list is filtered accordingly.




    Cost Report

    Cost Report

    Here, depending on settings in the control mask, the cost report for the selected employees is displayed.

    NOTE:
    (see control mask "Set display criteria for the cost report")



    Field Descriptions

    Day:

    Full day of the week name.

    Date:

    Used to enter or display a date of the day. If a date entry is required, the default value (today's date) is usually already preset.

    Time:

    Used to enter or display a time.

    Project:

    This field displays the assigned project.

    Serv.-Type:

    The activity type in cost accounting is a reference value that indicates the qualitative technical description of the production activity. Here, it can be classified between productive (=value-adding) and non-productive types.

    Plan.Hrs.:

    Indicates how many hours an employee must work in a given period of time

    Act.Hrs.:

    Indicates how many hours an employee actually worked in a given time period.

    Delta:

    The delta hours are the difference between actual and target hours. They indicate how many hours the person has worked more or less than contractually agreed in the specified period (= overtime or minus hours).

    PP Net:

    This is the monetary value, the amount of which shows the financial expenditure for the company per hour of use of a worker or a material (counterpart to VK=sales price).

    Sum PP:

    Specifies whether the material always incurs costs (fixed) or only when used (variable).

    VAT:

    VAT on the total price of certain items.

    Gross:

    Invoice costs including statutory value added tax

    Inv.Hrs.:

    This field contains the billable hours for an entered activity.

    SP Net:

    This is the monetary value, the amount of which shows the financial return for the company per day of use of a worker or a material (counterpart to EK=purchase price).

    Sum SP:

    Yield = Number of hours to be billed x VK

    U-Margin:

    Consumption margin for a project, i.e. the difference between ext. consumed budget and int. consumed budget.

    Rel.:

    The release indicates whether the record has been released for settlement.

    Clear:

    Specifies whether or not to display offsetting entries.

    Inv.:

    Specifies whether or not to display billed entries.

    Link:

    This field is optional. If a link is entered, the automatic user rights query is bypassed and the Trans_ID field is cleared.
