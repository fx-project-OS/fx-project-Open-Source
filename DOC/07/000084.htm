View of a project as project structure plan


Displays the structure of a complete parent project top-down, i.e. listed from top to bottom. The project type, the project status and the evaluation characteristics can be seen at a glance by means of frames, lines and colors. If desired, the critical path is also displayed.

A variety of selection criteria such as "client", "project manager", "project manager", "project status", "time period", etc. can be defined as filter factors.




Project selection for report

Project selection for report

Through various fields it is possible for you to limit the number of projects in a report, for example by only wanting to see all active projects, or only projects of a project manager or similar.

NOTE:
For additional information, please see the help of the respective report in the input mask.



Field Descriptions

Active:

Selecting this option displays all currently active projects.

Projects From:

By setting a date at this point, only projects that were started or planned AFTER this date will be displayed.

Main Project:

Top level of a project structure, under which all subprojects and tasks are subordinated.

Project:

In Project the project or task is selected for which the key data should be displayed.

Inactive:

Selecting this option displays all currently inactive projects.

Projects to:

By setting a date at this point, only projects whose planned or actual end date is before this date will be displayed.

Customer:

Here you can select the sold-to party of the task (project, subproject or task).

Planned:

Selecting this option displays all currently scheduled projects.

Displ. Tasks:

Here you can define whether tasks (and possibly to-dos) should also be displayed in the project selection list or not. If this checkbox is not set, only projects and subprojects are displayed.

To SubLevel:

Number of buffer days between two consecutive tasks to be used as a basis for determining the critical path.

Project Manag.:

Here the project manager of the task (project, subproject or task) is selected.

Completed:

Selecting this option displays all currently completed projects.

Max. Prj.:

Definition of the maximum number of text characters to be displayed for the project designation, i.e. if a text is too long (in particular for print formatting), the text is wrapped or truncated from the number of characters entered, depending on the "Line break" field set. If a value is entered, the project is also displayed without the project numbering.

An entry of '0' means: No text wrapping and project numbering (= default).

Project History:

If this checkbox is activated, then the critical path is also determined and displayed with red lines. The number of buffer days between two consecutive tasks serves as a basis here.

1st Project L.:

Here the project manager of the activity (project, subproject or task) is selected.

Critical:

If this checkbox is selected, only critical projects are displayed.

No Number:

If unchecked, the transaction numbers are not displayed.

Rec. Tasks:

If this field is checked, only displayed tasks are included and costed in the higher-level projects.

If this field is unchecked, all displayable tasks are included in the parent projects and costed, even if they are not displayed.
Displayable tasks are tasks that match the global project filter criteria.

Max. Chars:

In this field, the maximum number of text characters to be displayed can be entered, i.e. if a text is too long, the text will be truncated from the entered number of characters.
If this field is empty, no text truncation is performed, if the value is zero, no project name is displayed.
An input of two numbers separated by comma (e.g. 120, 30) causes the text to be shortened and wrapped (here 120 characters in display, wrapping after every 30 characters).

Detailed:

By ticking this checkbox, the normal report is extended by additional, more detailed information.

Pro Rata:

If checked, the planned and target values are adjusted proportionally to the period to be displayed.

Example: For a plan budget of 1,000.00 Euro in a 2-year period, this value is displayed as 500.00 Euro if the period to be displayed is exactly half of the plan period.

Skill Group:

In this field, the individual employee skill groups can be selected, i.e. the employee areas they cover. The selected group ID is stored internally.

Costs:

If this field is checked, the cost fields are shown in the project report.

Effort:

If this field is checked, the effort fields are shown in the project report.

Department:

Indicates the department in which the employee works.

Text Lines:

Text line


Accept

Clicking this button saves the entered data and continues the program execution.




Project structure plan

Project structure plan

The work breakdown structure allows you to view and analyze the structure of a complete parent project in detail.

Using the traffic light characteristic background colors, the complete project status can be determined at first glance.
The following criteria are checked to determine the background color: project status, time period, effort and budget.


Color legend:
White background, ie.i.e. project is still in the future.
Gray background, meaning project is either inactive (dotted border) or already completed (compact border).
Green background = OK, ie. Project is on schedule, effort-wise and budget-wise.
Yellow background = hint, i.e. at least one category (time, effort or budget) is in danger of jeopardizing the project and should be checked.
Red background = Attention, ie. at least one category (time, effort, or budget) jeopardizes the project and needs to be adjusted.
Framework legend:
compact, meaning project is either active or closed (gray background).
dotted, ie. Project is either inactive (gray background) or scheduled.
Line legend:

Line from bottom left to top right, meaning the project has already been started.
Two crossed lines, meaning the project has been completed.
Red lines, meaning the project belongs to the critical path.
