Administration > Database > Database Backup

Save or restore your own complete client data


The database backup is used to save (= backup) and restore (= restore) the client's own data. A restore can only be made to the same version that was active at the time of the backup. Licensing information is excluded from this. Furthermore, obsolete database backups can be deleted, and current ones can be displayed or downloaded in the browser.




Database backups

Listing of all database backups of the current client.

Depending on the action selected, the database can be backed up, restored or a backup file deleted.



Field Descriptions

Only Cust. Data:

Activate or deactivate an entry by placing a tick in the "checkbox".

Table Dis.:

If the checkbox is activated, all data required for graphical processing will also be listed in a report, which can be displayed and downloaded as a PDF including graphics.

Version:

The version number is incremented by 0.001 when a changed document is saved normally, and by 1 when a version change is explicitly requested or when the floating point range overflows (for 999 changes without a version change).

Manually changed versions are saved as a new file and activated, the old version is marked as inactive.

File Name:

The name of the generated file must be entered in this field.

Date:

Used to enter or display a date of the day. If a date entry is required, the default value (today's date) is usually already preset.

Number:

Number of entries

Size:

This field indicates the physical size of the file. With this information you can estimate, for example, how long a download would take with the usual bandwidth (-> Internet access).

Link:

This field is optional. If a link is entered, the automatic user rights query is bypassed and the Trans_ID field is cleared.


Save

Button to save data.

Delete

After pressing the button Delete the selected records will be deleted. If the record is still needed, you will receive an error message.
