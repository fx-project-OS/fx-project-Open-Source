Time recording for individual tasks of a day


Time recording is used for daily recording of hours and activities. This determines the current progress of the tasks, as well as the actual actual consumption. Time recording can be billed separately for hour-based tasks. Furthermore, a current hours status can be viewed at any time via detailed daily, weekly, monthly or yearly lists, in order to illustrate overtime, floating days, caps, etc.




Person selection

Selection of one or more persons from a list depending on the program function.

To create or edit persons, switch to the corresponding program function under the main menu item Master data.



Field Descriptions

Person:

This selection field is used to select the person whose data is to be edited or displayed depending on the action.


Select

A click on this button confirms the selection of the person and loads the corresponding data for editing or viewing.




Daily overview

Time data listing for a selected day

In the Daily overview, the recorded hours of a person for a selected day are listed in chronological order.

Entries can be changed or deleted by clicking on the corresponding symbol at the right edge of the line. New entries as well as changes are made in the corresponding edit mask.

With the help of the "Calendar Selection" on the right side it is possible to change to another Daily overview, a Weekly/monthly overview or Annual overview.



Field Descriptions

Time Data:

Start and end times of the recorded work, if they have been entered. A totals symbol is displayed for time total records.

Break:

Number of work break performed in time format during this time period.

Tasks:

Number and name of the task on which the time recording was booked.

Serv.-Type:

Subgroup of the task to which the time recording was posted.

Activity:

Detailed description of the performed work steps.

Status:

Release status of the posting record, where means:
· x (not yet released),
· MA (released by employee),
· PL (released by project manager) and
· VW/GL (released by administration or. Management approved).

An additional R indicates that this time entry has already been accounted for.

TSum:

Number of recorded hours in time format.

Display:

Clicking on the icon displays detailed information about this time tracking record in a new window.

Edit:

By clicking on the icon this time recording record is displayed for editing in the hours input mask.
If this icon is grayed out, this means that the record can no longer be changed. It may have already been billed or have a higher release status.

Delete:

By clicking on the icon this time recording record is deleted and at the same time preassigned for a possible new creation in the hours input mask. (Thus, a possibly accidentally deleted record can be created again and the deletion thus undone.)
If this icon is grayed out, this means that the record can not be deleted. Possibly it was already settled or has a higher release status.




Expense recording

Entering the time data

In Expense recording the user enters his hours for the selected day or changes selected hourly rates.

Already worked hours for the selected day he sees in Daily overview, from where he can also select hours for editing.

To switch to Weekly/monthly overview or Annual overview, use the "Calendar Selection" on the right.

The project manager can be informed if a certain project is already finished, if the hours worked are billable or which delays are to be expected or have occurred.

Hours can only be recorded for active tasks to which the person has been assigned.

Changes can only be made as long as the record has not yet been allocated.

"Unknown": on this project employees can write their hours if there is no suitable project.

"Floating day": this project gives the user the possibility to record his floating day.

Via the settings it can be set client-wide whether start and end time or only the hourly effort should be saved.

If two of the three fields are filled in, the third is automatically calculated when writing to the database.

NOTE:
The entries for the field "Project" can be administrated at the menue item"Define Projects".
The entries for the field "Release Status" can be administrated at the menue item"Controlling/Approve Time-Recordings".

Field Descriptions

Selection of the activity (project, subproject, task) for which you are authoriz:

Select and define the task to which hours are to be booked. If no suitable task is available, the bookings can also be made to [Unknown] and changed later.

Date:

Input field for the current date for which time recording entries are to be made. By default, this defaults to today's date.

CW:

Number of the week in which the day of the current time recording record lies.

Customer:

Client of the selected task. This entry can be left empty if a task has been selected, as it will then be automatically adjusted correctly when saving. For a booking on [Unknown] on the other hand, the client should be specified in order to be able to handle possible later time transfers more easily.

Release:

The release indicates whether the record has been released for settlement.

Deviation:

Variances indicate the additional effort by how many hours a project is delayed.

Alias field for time unit:

The time unit indicates whether the project is delayed by minutes, hours, man-days, man-weeks, man-months or man-years.

Rel. DOC:

The "Relative POC" depends on the determination method stored for the selected task and is used to determine the progress of the project.

Method:

Determination method of the degree of completion, which was deposited with the selected task.

TR n. invoice.:

Defines whether a time recording record is billable or not.

Completed:

Defines whether all work for a task is already done or not. (= equivalent to a project progress of 100%).
The responsible project manager receives an email about the completion of the task, as well as a date in the control center, where he can indicate whether the task is actually completed. Depending on his decision, the task is set to the status "completed" or remains active, and the employee is notified by e-mail that there are still improvements to be made.
If the project manager completes the task himself via time recording, no e-mail is sent or an appointment is created.

Start Time:

In start time is the start time of the worked activity.

End Time:

End time indicates the end time of the worked activity.

Time Sum:

Time spent in time format with 24-hour day limit.

Break:

Number of work break performed in time format during this time period.

Activity:

Detailed description of the performed work steps.

Annotation:

Internal additional information on the work steps performed, which should not be displayed in the performance record or similar.


Save

Clicking on this button saves the changes.




Weekly/monthly overview

Time data listing for a selected week or month

In the Weekly/monthly overview the recorded hours, as well as the absence days of a person or the day designations for a selected week or for a selected month are listed according to days.

By clicking on the edit or add icon on the right side of each day row, you can switch to the corresponding Daily overview.

The "Calendar Selection" on the right side can also be used to switch to Daily overview, another Weekly/monthly overview or Annual overview.



Field Descriptions

Day of Week:

Full day of the week name.

Date:

Input field for the current date for which time recording entries are to be made. By default, this defaults to today's date.

Activity:

Detailed description of the performed work steps.

H-Chk:

Traffic light graphic, which shows whether there is an hours recording for this day or not.

Plan.Hrs.:

The target hours indicate how many hours on average the person is contractually expected to work in the specified period.

Act.Hrs.:

The actual hours indicate how many hours the person actually worked in the specified period.

Delta-Val.:

The delta hours are the difference between actual and target hours. They indicate how many hours the person has worked more or less than contractually agreed in the specified period (= overtime or minus hours).

Edit:

Clicking on the or icon will switch to the corresponding Daily overview.




Annual overview

Time data listing for a selected year

The Annual overview lists a person's recorded hours and days of absence by month for a selected year.

By clicking on the arrow symbol at the right edge of each month line, you can switch to the corresponding Weekly/monthly overview.

The "Calendar Selection" on the right can be used to go to Daily overview, also to Weekly/monthly overview or to another Annual overview.



Field Descriptions

Name of Month:

Full name of the month.

Days:

Total number of days this month.

Workday:

Number of working days in this month.

Leave:

Number of vacation days in this month.

Sp. Leave:

Number of special leave days in this month.

Training:

Number of training days in this month.

Sick:

Number of sick days in this month.

Absent:

Number of unspecified absence days in this month.

Flextime:

Number of floating days in this month.

Plan.Hrs.:

The target hours indicate how many hours on average the person is contractually expected to work in the specified period.

Act.Hrs.:

The actual hours indicate how many hours the person actually worked in the specified period.

Delta-Val.:

The delta hours are the difference between actual and target hours. They indicate how many hours the person has worked more or less than contractually agreed in the specified period (= overtime or minus hours).

Edit:

Clicking on the icon will switch to the corresponding Weekly/monthly overview.
