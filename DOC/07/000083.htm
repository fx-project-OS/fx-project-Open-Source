Execution > Times > TR (Week/Month)

Time recording of selected tasks over a period of 1 week or 1 month


Time recording on a weekly or monthly basis is used to quickly record hours and activities over a longer period of time for the same tasks.

NOTE:
When saving the data records, the validation (check) and database routine of the program function Execution>Times>TR (Day) is called up internally. For this reason, an activation of this program function is a basic requirement to be able to perform weekly or monthly bookings.




Selection

The data entered in the selection mask is used to restrict and define the input mask to be displayed.

Person:
Here the person must be selected, to which the time recording is to be made. Normally only the own person is preset here, except the registered person has the employee roles "management" or "administration". With this role also other person data can be accessed and the input for these persons can be done.

Display Tasks:
Multiple selection field in which it is determined which tasks are to be listed for new creation in the input mask.

Customer (Client):
A client can be selected here, which will be set to [Unknown] in all newly created time entries. Useful if it is defined client-wide that clients have to be specified.

Activity
A text can be entered here, which will be preset in all newly created time entries. Useful if it is defined client-wide that activities have to be specified.

Date or CW:
Date or calendar week, which form the basis for the period to be displayed. If both values are incompatible with each other, then the last entry specified is used. If both entries were made at the same time, then the date entry has priority.

View:
Sets the period to be displayed, which contains the date value. The following options are available:

  • five days: corresponding to a working week from Monday to Friday
  • seven days: corresponding to a calendar week from Monday to Sunday
  • one month: corresponding to a complete month
.

Detailed Information:
If this checkbox is ticked (= default setting), then additional columns with information about the task are displayed in the input mask (such as target hours, actual hours, period or priority).

Display or Today:
When clicking on the button Display or Today the input mask is called up and prefilled with the corresponding data. If you click on Today, any date entries will be overwritten with today's date. If there are entries from an already existing input mask that have not yet been saved, a warning message is displayed.



Field Descriptions

Person:

This selection field is used to select the person whose data is to be edited or displayed depending on the action.

Project:

The Project field lists the tasks (projects or tasks) for which you have the appropriate rights and the corresponding filter criteria apply.

Customer:

Here you can select the client of the activity (project, subproject or task).

Activity:

In this field you have the possibility to describe your work in detail. The more exact the descriptions, the more exactly the work progress can be reconstructed at a later time.

Date:

Used to enter or display a date of the day. If a date entry is required, the default value (today's date) is usually already preset.

CW:

Number of the week in which the day of the current time recording record lies.

View:

In the field "View" you can specify how many days should be displayed on the calendar, e.g. one-day view or 7-day week etc.

Detailed Inf.:

By ticking this checkbox, the normal report is extended by additional, more detailed information.


Display

By clicking on this button, the list is generated and displayed. Depending on the defined selection criteria, the list is filtered accordingly.

Today

Button of the date selection. After clicking you will find yourself at today's date.




Enter

In the input mask all existing time registrations of the defined period are listed with the possibility to change or delete them (exception: time registrations which have already been accounted).

Subsequently, all selected tasks as well as the special case [Unknown] are listed in a separate line each in order to be able to execute new entries.

If the period is set to month and a lot of tasks are listed, so that they do not fit in the display area, you can move them to the left and right as well as up and down using scroll bars. The task, headings or totals part always remains in the display.

The left section contains task information adapted to the selected person, i.e. their target hours according to the percentage resource planning, actual hours, etc.

The right section is the actual input section. For each day, a time recording can be performed here for exactly one task, in these four standard fields:

  • Start corresponding to the start time, e.g. 10:00.
  • End corresponding to the end time, e.g. 15:00 o'clock.
  • Break corresponding to the work break, e.g. 0:30 h (= 30 minutes)
  • Duration corresponding to the work duration, e.g. 4:50 h


The logical relations between the fields are calculated by JavaScript, i.e. if a start time and an end time are entered, the duration is automatically calculated and displayed immediately, etc.

The footer displays day information, which is also automatically calculated by JavaScript. These are:
  • Total break corresponding to the sum of all work breaks of this day
  • Total duration corresponding to the sum of all working hours of this day
  • Target hours per day corresponding to the employee-specific target hours per day. the employee-specific target hours of this day
  • Difference target hours corresponding to the difference duration - target hours


When clicking on the Save button, the data is validated and written to the database by indirectly calling the "normal" time entry on a daily basis for each record.

NOTE:
Clicking on the date in the day column will jump to the "normal" time entry on a daily basis for that day. If there are any unsaved entries a warning message will be displayed.


Clicking with the right mouse button in one of the input fields will open an action menu in which further functionalities are provided, such as:
  • Edit entry: Opens a new window to be able to enter additional information for this time entry such as activity, note, billable, degree of completion etc.
  • Delete entry: Deletes the currently selected entries of the new entry from the list or marks an already existing time entry for deletion.
  • Delete day: like Delete entry, but for the whole day
