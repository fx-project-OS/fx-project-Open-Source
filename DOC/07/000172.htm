Creation > Contacts > Contact Person

Contact person of a contacted company


Management (new creation, modification or deletion) of contact data of contacts such as name, gender, salutation, identification number, personal picture, address or communication data.




Selection

In this sub-menu the person/company name to be edited is selected. If it is a program function for employee data (ext./int.) the selection field Inactive is additionally activated for selection. If the field Inactive is activated, also person groups which are no longer active are displayed.



Field Descriptions

Person:

blank

Sort Communication:

Setting how to sort the communication data to be displayed.


Select

A click on this button confirms the selection of the person and loads the corresponding data for editing or viewing.




Summary of information

People Overview

In the mask Summary of information you get a complete overview of the over- and under-organized persons.



Field Descriptions

Overview:

Overview gives a short overview of the selected person.




Definition

People management

Definition stores the most important data for a person as well as their assignment. Field Job Description is only a definition without special rights for data access and can be changed in program function Administration>Client>Reference Values with Job Description. Project managers with corresponding rights etc. are defined in the program function Creation>Employees>Skills/Abilities! Field Department can be managed in program function Administration>Departments>Department Administration.



Field Descriptions

Salutation:

In Administration>Skills>Skills the salutation of a person can be selected.

Sex:

In this field the gender of a person is selected, e.g. "male" or "female".

Image:

Assigned image in PNG, JPG, GIF or BMP format.

Name:

Control>General>Control Center contains the name of the person.

Customer (Client):

Here you can select the client of the activity (project, subproject or task).

First Name:

First Name stores the first name of a person.

Department:

Indicates the department in which the employee works.

Name affix:

Administration>Skills>Skill Groups is an extension of the field Control>General>Control Center. Here additional information can be stored, which possibly had no place in the upper field.

Job Description:

In Job Description the job description under which the person has been hired is selected.

Identif. No.:

Identification No. stores initials, customer numbers, or any other type of number for a person.

Title:

In Administration>Users>User Settings the title of a person can be selected.

Allocated to:

Allocated to Object included the parent.

Valid from:

Date from which a record is valid.

Valid to:

Date until when a record is valid.

Inactive:

A check mark in Department indicates people who should be hidden (employees who have left, old customers, ...).
Checked in Selection, the field also causes the inactive persons to be displayed with the addition "inactive".

Inactivating a client also causes it to be completely locked and no one can log in anymore.


Save

By clicking on this button, the changes are made. The text of the button depends on the current action mode.

Delete Add.

Delete additional data (= address and communication data) of the person/company

By clicking on the button Delete Add. Data all selected additional data, i.e. address and communication data of the person/company will be deleted.

Delete Company

Delete entire person or company, including all additional data.

By clicking on the button Delete Person the entire selected person or company including their additional data such as pictures, address and communication data will be deleted.
A person or company cannot be deleted if there are still subordinate persons to it.




Addresses

Address management

The submask Addresses stores addresses for the selected person/company. The number of empty entries can be determined by the program function Administration>Users>User Settings. The different types of addresses are maintained in Administration>Client>Reference Values with Address Type.



Field Descriptions

Only Cust. Data:

A check mark in Checkbox indicates that this entry should be deleted.

Address Type:

In Address Type you select the address type of the address.

State:

State contains the state of the address.

Street:

Street contains street and house number of the address.

Cntry:

Country Code contains the country code of the address.

Code:

Creation>Contractors>Roles contains the postal code of the address.

Place:

City/Town contains the location of the address.




Communication

Communication management

In mask Communication all relevant information on how to reach a person (except the address) can be stored. New types are maintained in Administration>Client>Reference Values with Communication Type.



Field Descriptions

Only Cust. Data:

A check mark in Checkbox indicates that this entry should be deleted.

Communicat.Type:

In Communication Type, the communication type is selected. "Default" types are displayed in Summary of information.

Number:

Number contains the communication number (phone number, e-mail address, ...).

Annotation:

Annotation contains additional information about the number (validity, availability times, ...).
