Transfer recorded time data to other tasks


Transfer of time recording entries from one task to another. Mass transfers are also possible.




Determine selection criteria

Define selection criteria for time account changes

Here, the employees' time account entries (possibly posted as unknown) can be assigned to a task over a certain period of time.



Field Descriptions

Indicates whether a material is purchased separately for a project.:

Indicates if a material is purchased extra "for (a) project". The acquisition cost is added to the internal consumed budget, the sales price is added to the external consumed budget of the project and its parents.

If no project is selected, these costs do not flow into the project.

Act. Start:

Indicates the start date of the respective reference object, e.g. projects, time data, travel data, etc.

Selection of the activity (project, subproject, task) for which you are authoriz:

The Project field lists the tasks (projects or tasks) for which you have the appropriate rights and the corresponding filter criteria apply.

Act. End:

The actual end date of the respective reference object, e.g. project, time data, travel data, etc. is displayed or specified here.

Person:

This selection field is used to select the person whose data is to be edited or displayed depending on the action.

Max. Chars:

Defines the maximum number of text characters to be displayed, i.e. if a text is too long (especially for print formatting), the text will be wrapped or truncated after the entered number of characters, depending on the "Line break" field set.

An entry of '0' means: no text wrapping. For print formatting, e.g., for the "Proof of service", the preset value of 50 (= default) is recommended.

Customer:

Here you can select the client of the activity (project, subproject or task).

Line Break:

Specifies whether texts are to be wrapped or truncated after a certain number of characters. When text is wrapped, the text remaining after a certain number of characters is continued in the next line and possibly wrapped again, and so on.


Display

By clicking on this button, the list is generated and displayed. Depending on the defined selection criteria, the list is filtered accordingly.




List of time account entries

List of time account entries

Displays all time account entries that match the selection criteria.

With the help of this mask it is possible to book time account entries from one project to another. The prerequisite for this is that the person is assigned as a resource in both projects and the "source project" contains time data of this employee.



Field Descriptions

Only Cust. Data:

Activate or deactivate an entry by placing a tick in the "checkbox".

Date:

Used to enter or display a date of the day. If a date entry is required, the default value (today's date) is usually already preset.

Time:

In this field, a selection menu can be used to select how fields of the type "Time span" are to be displayed or converted. Possible values are: "hour(s)", "day(s)", "month(s)" and "year(s)".

Proj. No.:

This is the directory number for a project.

The directory number cannot be changed for parent projects.

The directory number is automatically adjusted when deleting or moving.

By manually changing the directory number, you can move projects within a hierarchy.

Serv.-Type:

The activity type in cost accounting is a reference value that indicates the qualitative technical description of the production activity. Here, it can be classified between productive (=value-adding) and non-productive types.

Effort:

If this field is checked, the effort fields are shown in the project report.

Project:

In Project/Task the project or task is selected for which the key data should be displayed.

Activity:

In this field you have the possibility to describe your work in detail. The more exact the descriptions, the more exactly the work progress can be reconstructed at a later time.

Time:

In time spent the user can specify the time spent in hours and minutes.

TR ni:

Here you can specify whether the time recording should be invoiced or not.

Inv.:

Specifies the type of allocation.

Rel.:

The release indicates whether the record has been released for settlement.

Customer:

In customer the client of the project or subproject or task is selected.

Prs-Account.Ap.:

Specifies the type of billing according to which person services are billed, for example, fixed price, person days, hours, etc.

Depending on which value is set here, this has an influence on the invoice writing.
Fixed Price: In this case, the value entered in the Intern Plan. Budget field of this project will be used as the respective fixed price amount in the invoice writing.
Per Hour Basis: Invoice items are created based on hours worked (or travel expenses recorded).
Not Invoiceable: Tasks that carry this accounting type will not be included in the invoice writing for person fixed price or person hourly basis.


Multi Function
