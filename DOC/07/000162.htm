Reporting > Projects > MTA (Milestone-Trend-Analysis)

Graphical representation of the milestone date changes of a project during the planning process


The MTA (Milestone Trend Analysis) is a method for monitoring current project progress.

The MTA helps to better control or estimate the schedule progress of the projects. With the MTA it is possible to detect schedule shifts (delays or completions) at an early stage in order to be able to react accordingly. The prerequisite for the MTA are firmly defined milestone dates.




Control screen for selecting the display criteria

Control screen for selecting display criteria for the MTA (Milestone Trend Analysis) graph.

If a project date is changed, the new date on the change date (= report date) is defined as the new target date (= milestone date). This is entered in the MTA accordingly. If not, the old date is retained. In the course of a project, there are usually many such date shifts. Therefore, a corresponding number of new target dates (that is, the milestone date is moved back or forward) are collected for each milestone. These dates result in a trend for each milestone. This is visualized by means of the graphical evaluation of the MTA.

Based on the curve, a trend can be calculated whether, and if so, where the deadline will most likely shift.

A distinction is made between four basic principles:


  • A horizontal trend means no deadlines have been postponed (yet), i.e. the deadline will be met as planned.

  • A downward trend indicates that the planned date will be reached earlier, i.e. subsequently planned projects can be brought forward.

  • An upward trend indicates that the planned deadline cannot be met. Subsequent projects and possibly the whole project are in danger and have to be postponed.

  • A zigzag course indicates that there are uncertainties in the planning of the target dates. It cannot be estimated whether the deadline can be met or not. Possibly the whole project must be planned again.

Field Descriptions

Selection of the activity (project, subproject, task) for which you are authoriz:

In Project, the task (project or task) for which the milestone trend analysis is to be created is selected.

Time Span from:

This field displays the point in time from which data for a project should be listed.

In input mode, a date can be entered here according to the user settings.

Displ. Sub Projects:

If the checkbox is activated, all subordinate subprojects and tasks are included in addition to the main project.

Attention!
This can lead to longer waiting times for large projects.

Accuracy:

Here you can define how many sections the actual time period should be divided into, i.e. the number of reporting points can be defined here. The higher the number the more subdivisions, the more detailed the line. For example, if a project goes over 365 days (1 year) and you enter 12 in this field, the period will be divided into 12 sections of 30-31 days each (= 365/12) and you will thus get one report time for each month. If this field remains empty or 0 is entered here, then a point is generated in the graphic for each individually determined report time of a milestone (= default setting).

Time Span to:

This field displays the date up to which the data for a project should be listed.

In input mode, a date can be entered here according to the user settings.

Table Display:

If the checkbox is activated, all data required for graphical processing will also be listed in a report, which can be displayed and downloaded as a PDF including graphics.

Size:

Here you can define how many pixels wide or high the MTA triangle should be. The value can be between 400 and 1200 pixels. The default value is 500 pixels.


Display

Graphically display all projects that match the above selection criteria.




Data report for the preparation of the graphic

Data for the preparation of the MTA in report form incl. graphics

This display corresponds to the tabular formatting of the MTA graphic. Each milestone that is displayed in the graphic is also listed here. The individual actual dates (= report dates) are numbered consecutively and displayed with the corresponding planned target dates (= milestone dates). The difference to the originally planned milestone date (= date next to milestone name) results in the shift in days, which is visually displayed in the MTA.

On the basis of the curve, the estimated end date for projects that have not yet been completed is displayed as a result, as well as the actual end date for completed projects.

End dates are always on the hypotenuse, with completed end dates shown with a border and estimated end dates in brackets. Estimated end dates can also be displayed as a line on the hypotenuse, depending on the curve progression, as the two end points are always calculated, once from the end point based on the curve progression and once as a horizontal line if the last milestone date reached is retained.

Field Descriptions

No.:

Sequential numbering of each report point per milestone.

Report dates:

A report date corresponds to the actual change date of a milestone target date. If a fixed precision was entered in the selection mask, each report date corresponds to a section of the total period divided by the precision.

Milestone dates:

Target date of the milestone at the report time.

Delta-Difference:

Difference in days of the current target date (milestone date) to the original target date (milestone date). A deadline delay is represented by a positive difference, whereas an early achievement of a deadline is represented by a negative difference.

Icon:

Graphical display of the difference of the target dates as a tendency arrow. A deadline delay is symbolized as a red arrow pointing upwards, whereas an early arrival of a deadline is symbolized as a green arrow pointing downwards. Equal values are symbolized as a green diamond.
