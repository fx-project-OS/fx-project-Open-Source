View of a project or subproject as Gantt chart


Displays all selected projects in a graphical format, where each project found that matches the filter criteria is displayed in a separate line. The temporal relationship of the projects to each other can be seen through the division of the columns, as each column represents its own time unit. This unit can be freely set between "days", "weeks", "months" or "years".

Through the use of graphic symbols, the project type as well as the project status can be seen at a glance. If available, the corresponding milestones are also displayed graphically in an extra line, including their dependencies on the start or end nodes of the assigned activity.

Problem projects (with time deviation or budget overrun) are also represented by a separate graphic symbol.

Display project overview graphically

The Gantt view is a graphical representation of one or more freely selectable projects.

Furthermore, you can define which period should be displayed and how it should be formatted (days, weeks, months or years), or whether only planned or also actual data should be displayed.

The display can also be limited to the project type, i.e. whether only projects, only tasks or everything should be displayed.



Field Descriptions

Project:

Selection of a project or of "All projects". The graphical Gantt chart will be created for the selected and all subordinate projects and tasks.

Project Type:

Selection which project type should be displayed in the Gantt plan, i.e. "All", "Projects only" or "Tasks only". If subordinate projects, tasks or milestones exist, the corresponding levels can be shown or hidden.

period of time:

Specify the time period (start and end date) in which found entries are to be displayed. Normally this period is preset with the last month.

CAUTION! If the period is long, this can lead to considerable waiting times.

Text:

Text field

Variable text field that can be customized programmatically.

Time Span to:

This field displays the date up to which the data for a project should be listed.

In input mode, a date can be entered here according to the user settings.

Autom. Time Span:

Serves to automatically preset the time period based on the selected project. The project target and actual periods as well as the milestone period are taken into account.

Status:

Used to limit the projects to be displayed based on the status, i.e. to display only "planned", "inactive", "active" or "completed" projects.

To SubLevel:

Number of maximum project levels to display, where means:

  • "0" or "empty" no restriction,
  • "1" parent projects only,
  • "2" parent projects and first sublevel,
  • "3" parent projects and two sublevels,
  • usw.

Only Planned:

Used to display only planned-target data, i.e. actual data from time recording, appointments, etc. are not taken into account.

Cal.Displa:

Used to specify the calendar time intervals per column, i.e. days, weeks, months or years.

If "Automatic" has been set, the calendar time interval is determined programmatically based on the length of the total period to be displayed.
