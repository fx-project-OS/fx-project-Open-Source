Overview of all existing users in the system


The user overview lists all defined users with the assigned profiles and used license points. Activated special programs are also listed.




User overview

Listing of all existing users including the following information:

  • User info
    = User icon and username. Clicking on the name will take you to the detailed view of the user.
  • Person Name
    = Each user has exactly one person assigned to it. This person name is shown here in "blue". However, several users can be assigned to one person, e.g. for vacation replacements, for logging in with different profiles, etc.
  • E-mail address
    = the main e-mail address stored directly with the user. The assigned person of the user may contain other email addresses (in communication data).
  • User IS the main administrator
    = Per client, there is exactly one main administrator - this is deposited as the contact person of the client and has special rights, as well as specially assigned administrator program functions.
  • User IS an administrator
    = The user has been assigned administrator rights. There can be several administrators per client - these have special rights, as well as specially assigned administrator program functions.
  • PersonType
    = indicates the person type of the assigned person, e.g. employee internal or external, client etc.. According to the person type, the user has different rights, e.g. a client can not enter a vacation - this is reserved for the employee (internal) or similar.
  • Logins
    = Number of total logins in fx-project, as well as the date of the last login.
  • Default language
    = the language specified in the user settings. The language can be changed by the user himself at any time - prerequisite is: The language special program has been activated and the user has been assigned to the corresponding language profile group.
  • Default group
    = Each user is assigned to a default profile group with various default program functions, e.g. "Change password" or "Set user settings". If the user is an administrator, he is also assigned to the profile group "Admin program" with further standard program functions, e.g. "Create user", etc. These standard program functions are directly linked to the standard profile groups and can not be deleted.
  • User profile groups
    = Each profile group is assigned to different program functions.
  • Employee role
    = The employee roles are responsible for assigning permissions, e.g..For example, the general manager has by default right to all people and projects, whereas a project manager has rights only to projects that have been assigned to him, as well as to people who are assigned to his projects.
