Reporting > Projects > DTA (Date-Trend-Analysis)

Graphical representation of the start and end date changes of a project during the planning process


The TTA (Schedule Trend Analysis) is a method for monitoring the current progress of a project.

The TTA helps to better control or estimate the schedule progress of the projects. With the TTA it is possible to detect schedule shifts (delays or completions) at an early stage in order to be able to react accordingly. In contrast to the MTA, no fixed milestone dates are required for this.




Control screen for selecting the display criteria

Control screen for selecting display criteria for the MTA (Milestone Trend Analysis) graph.

If a project date is changed, the new date on the change date (= report date) is defined as the new target date (= milestone date). This is entered in the MTA accordingly. If not, the old date is retained. In the course of a project, there are usually many such date shifts. Therefore, a corresponding number of new target dates (that is, the milestone date is moved back or forward) are collected for each milestone. These dates result in a trend for each milestone. This is visualized by means of the graphical evaluation of the MTA.

Based on the curve, a trend can be calculated whether, and if so, where the deadline will most likely shift.

A distinction is made between four basic principles:


  • A horizontal trend means no deadlines have been postponed (yet), i.e. the deadline will be met as planned.

  • A downward trend indicates that the planned date will be reached earlier, i.e. subsequently planned projects can be brought forward.

  • An upward trend indicates that the planned deadline cannot be met. Subsequent projects and possibly the whole project are in danger and have to be postponed.

  • A zigzag course indicates that there are uncertainties in the planning of the target dates. It cannot be estimated whether the deadline can be met or not. Possibly the whole project must be planned again.

Field Descriptions

Selection of the activity (project, subproject, task) for which you are authoriz:

The Project field lists the tasks (projects or tasks) for which you have the appropriate rights and the corresponding filter criteria apply.

Time Span from:

This field displays the point in time from which data for a project should be listed.

In input mode, a date can be entered here according to the user settings.

Displ. Sub Projects:

If the checkbox is activated, all subordinate subprojects and tasks are included in addition to the main project.

Attention!
This can lead to longer waiting times for large projects.

Accuracy:

Sets the accuracy of the display.

Time Span to:

This field displays the date up to which the data for a project should be listed.

In input mode, a date can be entered here according to the user settings.

Table Display:

If the checkbox is activated, all data required for graphical processing will also be listed in a report, which can be displayed and downloaded as a PDF including graphics.

Size:

This field indicates the physical size of the file. With this information you can estimate, for example, how long a download would take with the usual bandwidth (-> Internet access).

Data report for the preparation of the graphic

Data for formatting the TTA in report form incl. graphic.
This display corresponds to the tabular formatting of the TTA graphic. Each date that is displayed in the graphic is also listed here. The individual actual dates (= report dates) are numbered consecutively and displayed with the corresponding planned target dates. The difference to the originally planned end date results in the shift in days, which is displayed visually in the TTA.
Based on the curve, the estimated end date for projects that have not yet been completed is displayed as a result, as well as the actual end date for completed projects.
End dates are always on the hypotenuse, completed end dates are shown with a border and estimated end dates are shown in brackets. Estimated end dates can also be displayed as a line on the hypotenuse, depending on the curve progression, since the two end points are always calculated, once from the end point based on the curve progression and once as a horizontal line if the last end date reached is retained.

Field Descriptions

No.:

Sequential unique numbering.

Actual End Date:

The actual end date of the respective reference object, e.g. project, time data, travel data, etc. is displayed or specified here.

Project dates:

The target end date of the respective reference object, e.g. project, time data, trip data, etc. is displayed or specified here, i.e. by when the object should be completed according to planning.

Delta-Difference:

The delta hours are the difference between actual and target hours. They indicate how many hours the person has worked more or less than contractually agreed in the specified period (= overtime or minus hours).

Icon:

Specifies which icon is used with this entry.
