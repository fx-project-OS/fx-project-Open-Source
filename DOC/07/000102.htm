Material consumption recording of consumable or time materials for individual tasks of a day


Recording or listing the daily material consumption or material usage.




Material selection

Search criteria can be used to limit the number of material lists that are to be displayed or changed. As search criteria, for example, article numbers, material names, assigned projects, material types, usage types, storage locations and/or start or end dates can be specified.



Field Descriptions

Int. Artikelnummer:

If a number is entered here, only materials are listed which contain this number completely or partially.

Materialname:

If a name is entered here, only materials are listed which contain this name completely or partially.

Task:

The Project field lists the tasks (projects or tasks) for which you have the appropriate rights and the corresponding filter criteria apply.

Materialtyp:

If a type is selected here, only materials of that type are listed (material types are self-defined lookup values).

Lagerort:

If a location is selected here, only materials are listed which are stored at this location.

Verwendungsart:

Specify what type of material to list: Consumable and/or time material.

Search from:

In this search field you can define the validity start of the time frame of the search.

Search up to:

In this search field you can set the validity end of the time frame of the search.


Search

Clicking on this button starts the search.




Material allocation and consumption

Recording of the daily material consumption or the material usage of consumable or time materials.

Consumption recording can only be carried out for active tasks to which the selected person has been assigned in resource planning.

Changes can only be made as long as the consumption recording has not yet been allocated or released by a higher instance.

Entries can be posted to [Unknown] as long as, for example, no suitable task has been created.

To switch to the weekly, monthly or yearly overview the calendar of the navigation help on the right side can be used. By clicking on the "Today" icon in the calendar, the current week overview can be displayed and an entry can be made for today's date.



Field Descriptions

Material:

Selection of the consumption or time material for which a consumption or usage is to be recorded for this day.

If the desired material should not be listed due to the current selection criteria, the selection mask can be called again by a click on the [Back] button and adapted if necessary!

Captured for:

Selection of the person for whom the material entry is to be made.

The listed persons here depend on the one hand on the person authorization of the current person (= employee role etc.), and on the other hand on the authorization of the corresponding persons for a material consumption recording (= program assignment).

Task:

Selection of the task or [Unknown] on which the material entry is to be performed.

The tasks listed are dependent on the person selected and their active task assignment for that day based on resource scheduling.

Customer:

Selection of an optional sold-to party for a material entry on [Unknown].

This value is automatically adjusted to the client of a task when selecting a task.

Date:

Current date for which the material entry is to be made.

Normally this date is already correctly preset by the calendar navigation or by clicking on the icon of the overview list.

Start Time:

Start time of the material usage of the selected time material.

(This field is hidden for consumables!)

Text:

Text field

Variable text field that can be customized programmatically.

End Time:

End time of the material usage of the selected time material.

(This field is hidden for consumables!)

Hours:

Number of material usage hours of the selected time material. This value is calculated automatically as the difference between start and end time of the material usage.

(This field is hidden for consumables!)

Amount:

Number of material consumption (incl. unit) of the selected consumable material.

(This field is hidden for time materials!)

Dummy:

Dummy

Selling Unit:

In which unit is the quantity of items sold, e.g. kg, pieces, meters etc.?

ME n.verrech.:

Used to specify whether or not this material usage entry is billable.

Description:

Explanatory description, detail, remark or similar for this material consumption record.

Annotation:

Internal additional information for this material consumption recording, which should not be displayed in the service record or similar.


Save

By clicking on this button, the changes are made. The text of the button depends on the current action mode.




Material consumption list

This list displays all material consumption and usage within a specified time period.



Field Descriptions

Del.:

Checkbox to conveniently mark multiple material consumptions which should be deleted.

Day:

Weekday abbreviations, i.e. from "Mo" (= Monday) to "Su" (= Sunday).

Date:

Current date of the listed row.

Material Name:

Name of the material for which the material consumption was posted.

Tasks:

Number and name of the task for which the material consumption was booked.

Description:

Explanatory description, detail, remark or similar for this material consumption record.

Time:

In the case of a time material, the time span for which the material usage was booked is displayed here.

Quantity:

In the case of a consumable material, the number of consumed materials per unit that were posted is shown here.

Unit:

With a consumption material here the unit of the used materials stands which were booked.

Hr.-Costs:

Monetary value, the amount of which shows the financial expenditure for the company per hour of use of a worker or a material (counterpart to VK=sales price). Here the EK (purchase price) is to be entered, which the resource receives for the current project.

Example: An employee has a purchase price of 25 €. If he works on a "normal" project, his purchase price is also 25 €. But if he works on a project that needs all his experience, the price can be adjusted to this project (e.g. price 40 €). The adjustment is also possible for the sales price.

Hr.-Yield:

Monetary value, the amount of which shows the financial return for the company per day of use of a worker or a material (counterpart to EK=purchase price). The sales price that the resource generates for the current project must be entered here.

Example: An employee has a sales price of 40 €. If he works on a "normal" project, his sales price is also 40 €, but if he works on a project that requires all his experience, the sales price can be adjusted to this project (e.g. sales price 60 €). The adjustment is also possible for the EK.

U-Margin:

Consumption margin for a project, i.e. the difference between ext. consumed budget and int. consumed budget.

Info:

Information about this material consumption recording, ie:

person icon
display who made this capture. In the associated tooltip are extra information, as for whom and when this capture was performed including a detailed history, what was changed when.
Cancelable icon
Display whether this capture is billable or not.
billed icon
indicates whether this capture has already been billed or not.
released icon
indicates whether this capture has already been released or not.

Icon:

The individual icons for performing intentional actions, ie.
New creation icon
Create a new material consumption entry for that day.
Time icon
Directly switch to the time entry program function for that day.
Change icon
Open this material consumption recording in the detail view to possibly change.
Delete icon
Immediate deletion of this material consumption recording.
