Export projects from fx-project® as a MS-Project® CSV file


Generates a CSV file from the selected project or sub-project, which can be read into MS-Project®. This makes it very easy to exchange data between fx-project and MS-Project®.


NOTE:
The fields used in the CSV file can be freely defined and adapted. However, the default values are designed in such a way that a manual adjustment to MS-Project® is not necessary.




Export projects to MS-Project

Export projects to MS-Project

Here you can export projects from fx-project to MS-Project® (as a text file separated by tabs).

  • Select a project and specify in which format the date and in which unit the effort in MS-Project® is desired. Then click on the download button to download the file.
    The following values will be transferred to MS-Project®:
    • Number
    • Project name
    • Target start
    • Target-.End
    • Type of project
    • Successor
    • Target effort
    • Description
    • Priority
    • Performance
    • Item
    • Level
    • Date of creation
    .

    After exporting, open the file with MS-Project® and select an appropriate MS-Project® import mapping (or create a new file, see MS-Project® Help).



    Field Descriptions

    GIF-Picture:

    Specifies which icon is used with this entry.

    Selection of the activity (project, subproject, task) for which you are authoriz:

    The Project field lists the tasks (projects or tasks) for which you have the appropriate rights and the corresponding filter criteria apply.

    Dest. (File):

    Select destination (file, project, etc.)

    Sub Projects:

    If the checkbox is activated, all subordinate subprojects and tasks are included in addition to the main project.

    Attention!
    This can lead to longer waiting times for large projects.

    Separator:

    Separator for the export fields for project export in MS_Project.

    Date Format:

    Defines the format in which all dates are displayed and read in. The date must be defined with the following variables. Separator any:

    dd: Day 2 digits
    mm: Month 2 digits
    yy: Year 2 digits
    yyy: Year 4 digits

    Truncate Prefix:

    Prefix (German: Präfix or also Vorsilbe genannt) refers to a string of characters preceding the root of a word, here to create a group of words that belong together.

    It is used, for example, in the project name to achieve a uniform project affiliation already in the name. The placeholder structure to be used here can be defined in the administrator settings.

    Copy Resources:

    If this field is checked, the resources that are assigned to a task are also copied.

    Field Descriptions

    Status:

    This field shows how far a project has already progressed. In input mode, you can assign an attribute from a selection list to a project here, which reflects whether or how far a project is already advanced.

    The status can also be used to decide whether project data should be included in a report or not.

    Project:

    This field displays the project name of the currently selected project.

    Time Period:

    Displays the runtime of the project in a combined field.

    Plan. Eff.:

    The target effort corresponds to the planned time effort in hours, man-days or man-years, etc. The estimated effort is entered when defining tasks. For projects/sub-projects, this value corresponds to the sum of all subordinate tasks.

    Prio.:

    Indicates the urgency of the (sub)project.
    The higher the priority of a project, the higher it is displayed in the control panel of an employee.
