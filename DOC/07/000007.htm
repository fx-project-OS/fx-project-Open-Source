Contract data of an internal employee, such as entry/exit date, standard hourly rate, working hours, working days, annual leave, date of birth, etc.


Additional data for employees, which can be scheduled as resources, e.g. entry and exit date, target working hours per week, standard EC or CC, account details, etc. Furthermore, it is possible to determine on which days of the week an employee works or has time off, as well as their capping hours.

In this field, an entry must be selected from a selection of possible references to company (or branch) addresses.

This is the monetary value, the amount of which shows the financial expenditure for the company per hour of use of a worker or a material (counterpart to VK=sales price).

The entry date of a person, i.e. since when the employee has been employed by the company, must be entered in this field.

In the field, an entry is selected from a list of possible operational areas for the employee. This entry specifies in which federal states or countries the employee is available on a mobile basis.

SP:

This is the monetary value, the amount of which shows the financial return for the company per day of use of a worker or a material (counterpart to EK=purchase price).

Leave Date:

This field can be used to enter a person's leaving date, i.e. until when the employee was employed by the company.

This field displays the number of working hours to be worked by an employee (per week) and on which days the employee is available as a resource.

Bank Code:

The unique bank code of the employee's bank details can be entered in this field.

IBAN:

The IBAN (= International Bank Account Number) is an international, standardized notation for bank account numbers.

Composition of IBAN:

  1. 2-digit country code (consisting of letters)
    2. .
  2. 2-digit check digits (consisting of digits)
    3. .
  3. Max. 30-digit BBAN (= Basic Bank Account Number), i.e. the account identification (consisting of letters and/or digits)


The IBAN can comprise a maximum of 34 digits. A German IBAN always has exactly 22 digits, as it consists of the 2-digit country code DE, followed by a 2-digit check digit, followed by the 8-digit bank code, followed by the 10-digit account number. Shorter account numbers are extended to 10 digits with leading zeros.

Bank Account No.:

The account number of the employee's bank account is entered in this field.

BIC:

The BIC (= Bank Identifier Code or SWIFT address) is an internationally standardized code that uniquely identifies each directly or indirectly participating partner worldwide.

The BIC has a length of 8 or 11 alphanumeric characters with the following composition:

  1. 4-digit bank code (consisting of digits)
    2. .
  2. 2-digit country code (consisting of digits)

  3. 2-digit location coding (consisting of digits and/or numbers)

  4. 3-digit coding of the branch or department (consisting of digits and/or numbers)

Handicapped:

This field specifies whether the employee has a severe disability or not.

Flextime:

This field indicates whether the worker is entitled to work in floating day.

If the user is authorized to do so, he can book time records to "Floating day" within the time recording.

This allows the user to enter a floating day for e.g. Friday, if he has already fulfilled his target until Thursday.

By making this entry, the user is also prevented from being registered via the time recording check program, even though he has already worked the necessary hours.

Time Recording Check:

To control the daily time recording, a check program can run in the scheduler which checks whether the time recording has been carried out correctly.

If this value is set, the check is carried out in the scheduler and the employee receives a notification if there is no or an incorrect time recording.

If this value is not set, the check in the scheduler is not performed.

Time Restriction:

Specifies whether time restrictions per day are to be adhered to during time recording, i.e. if this value is set, the time recording hours per day may not exceed the set limit value defined for the client.

If this value is not set, the full 24 hours can be recorded daily.

Internal Annotation:

This field is used to annotate a project for internal purposes only.
