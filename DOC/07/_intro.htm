Introduction

fx-project is a 100% web-based project management and controlling system on internet/intranet basis. JavaScript elements are used to increase the user usability. This enables location-independent working for all project participants without prior installation of a program on a computer. A common browser is sufficient and the basis for a worldwide language-independent collaboration is created.




Why project management?


Recent studies (including Time Kontor) have shown that about 20% of projects fail, or are canceled (depending on the industry). If projects fail, this is mostly due to exceeded time and cost agreements. An important factor here are the resources in the form of qualified employees. Controlling is responsible for ensuring that deadlines can be met. fx-project supports project management in all necessary areas to ensure that a project reaches its planned goal in organisational terms.



Project management


When defining projects, all necessary information is recorded. In this process, a project can branch into any number of parts. Usually, a project consists of a mother project (= main project), which can be divided into any number of sub-projects (= collective projects or sub-projects). The subprojects can again be divided into tasks (= work packages), which represent the smallest project parts.





100% internet and intranet based


fx-project is 100% internet and intranet based. This means that users are independent of location and can check on the current status of their projects at any time.



Operating system independent


An enormous advantage of fx-project is that it is operating system independent.

(Server: Windows, LINUX or UNIX Sun Solaris, Client: any HTML 5.0 compatible browser)



Browser Independent


In addition to operating system independence, fx-project offers full browser independence. Whether Internet Explorer, Mozilla Firefox, Opera, Google Chrome or Apple Safari is used, fx-project is browser-independent and does not restrict the user's choice.



Database independent


fx-project comes standard for MS-SQL, PostgreSQL and Oracle.



Data security


Server-side SSL (Secure Socket Layer) encryption or VPN are, of course, available on request.



Multilingualism


Of course, a system that can be used worldwide must also have multilingualism, so fx-project. By default, fx-project is delivered in German and English. However, the entire system can be translated into other foreign languages in a short time.



Client capability


fx-project is multi-client capable. Each client is an independent self-contained system.



Data dictionary based information system


fx-project is built on a base system developed specifically for this purpose. All information is mapped centrally in a DataDictionary (DD) and is constantly in connection with the programs. Whether layout, report, help or documentation, the information always comes from the DD.

Program start and program end

fx-project is started via the link address in the browser that was provided to you by the administrator.

The user login mask is displayed on the start page. Enter the client no. assigned to you here, your username as well as your password.

If you have forgotten the password, contact your administrator to reset the password.

You can access the fx-project application (= login) by clicking on the [Login] button or by pressing the [Return] key .




To optionally exit the program (= logout), click the logout button in the top right navigation toolbar.

If you do not log out, fx-project remembers your login data for today, so you can continue working where you left off without logging in.

Program types

Basically, there are three different types of programs in fx-project: default programs, administrator programs and user programs.

1. Standard Programs.
Standard programs are program functions that are automatically assigned to each user without any special assignment of rights, such as user preference, password, filter, etc. This means at the same time that every new user needs at least as many license points as necessary for the standard programs.

2. Admin Programs.
Admin programs are special program functions that can only be used by the main administrator or a user with administrator rights, such as the new creation of clients, users or special programs to reset the password for a user.

3. User Programs.
User programs are program functions that can be assigned to any normal user, such as time tracking, control center, scheduler, etc.

General procedure for defining a project

In order to be able to create a project, some prerequisites have to be met so that the mandatory fields can be filled in during input:


Create client:

When defining the project, it is important that the principals have already been entered in the Creation>Customers>Basic Data program function. In addition, the contact person must be defined in Creation>Customers>Contact Person. Only if these requirements are met, a project can also be assigned to a principal.


Employee data:

In order to be able to plan a project completely, all employees involved in the project should be created in the program functions Creation>Employees>Basic Data or Creation>Contractors>Basic Data beforehand. Likewise, the settings regarding working time and availability should be correctly stored in the respective additional data.
The resource planning is then done either in the function Planning>Persons>Assign Person Resources (HR) or most simply directly in the graphical project planning Planning>Projects>Matrix: Graphically as Gantt.


Determine project manager, deputy project manager and project manager:

In order to select a project manager, the appropriate employee roles must be assigned to the desired individuals. To do this, first go to the program function Creation>Employees>Roles (for internal persons) or Creation>Contractors>Roles (for external persons) and assign a special "employee role" to the corresponding employee, in this case project manager or project manager.


In addition, special skills of the employees can be stored, so that employees with special knowledge can be found more quickly during resource planning. These skills are also stored in the project, in order to be able to make later appropriate evaluations on projects. In Administration>Skills>Skill Groups and Administration>Skills>Skills, the characteristics of these skills can be defined individually as a group and subgroup.


Creating a project:

Once the above requirements are met, the desired project can be defined in Creation>Projects>Create Main Project. In the second step, using the graphical input in Planning>Projects>Matrix: Graphically as Gantt, the project can be designed in the matrix, the effort can be planned, The project details can be set and the resources can be assigned.
