Reporting > Projects > EVA (Earned-Value-Analysis)

The Earned Value Analysis (EVA) or Performance Value Analysis (PVA) sets the current project progress in relation to the planned value to evaluate the work performed


Earned Value Analysis (EVA) or Performance Value Analysis (LWA) relates the current project progress to the planned value in order to evaluate the work performed.




Determine selection criteria

Specify the selection and display criteria for the Earned Value Analysis (EVA).

The determined data are prepared and displayed in a graphic chart and optionally in table form.



The displayed data is determined based on the selected tasks. The underlying tasks can be influenced by the following parameters:

    .
  • Project:
    A project must be selected from the complete project list. This list contains all project types (parent project, subproject or task) to which one has corresponding rights. When selecting parent or sub-projects, the underlying tasks are automatically expanded to include all tasks subordinate to this project. When selecting a task, only that task will be used as the base.

  • Mainly responsible persons:
    If main responsible persons such as client, project manager or project manager are selected, the underlying tasks are automatically narrowed down to the tasks the selected person is assigned to.

  • Project status:
    If one or more project statuses are checked, the underlying tasks are automatically narrowed down to the tasks that have the corresponding status. If no status is checked, or if all statuses are checked, no restriction is applied.




If the determined data should be displayed not only as a graphic diagram, but also in table form, then simply the checkbox next to the Table view field must be checked. The amount of information displayed can be specified with the Display Type selection box. The choices here are:

  • Automatic
    On the basis of the number of days in the determined period, the most suitable display type is determined. "Suitable" here means: the list should consist of a maximum of 60 information lines.

  • Day(s)
    Each day of the determined time period is displayed.

  • Week(s)
    Only the first day of each week in the determined time period is displayed.

  • Month(s)
    Only the first day of each month (= first of the month) in the determined period is displayed.

  • Quarter(s)
    Only the first day of each quarter in the determined period is displayed.

  • Year(s)
    Only the first of January of each year in the determined time period is displayed.



NOTE:
The table always displays the first and last day of the determined period.
The graph chart is always based on day-exact data, i.e. the restriction of the display type has no effect on the chart, but only on the rows of the table to be displayed.

Graphic diagram

Display graphs of Earned Value Analysis (EVA) and optional tag-specific tables.

The following data is determined and displayed:


  • Day/Date
    The abbreviation of the day of the week and the date of the determined tag-exact data.
    • .
  • Target effort - Planning effort in hours
    Planning effort is determined when a project is defined and corresponds to the summed planned effort of all subordinate tasks at a given point in time.

  • Effort. Actual - Hourly effort expended
    The hourly effort expended on a project corresponds to the summed incurred time tracking, travel, and material hours of all subordinate tasks at a given point in time.

  • S/I - Degree of Completion Target/Actual Comparison
    Indicates the percentage of a project that has been completed. The planning effort is put in relation to the hours worked.
    • .
  • PC - Planned Cost (= Planned Cost "PC")
    Planned costs are defined when a project is defined and correspond to the summed planned total internal budget (= people budget + material budget) of all subordinate tasks at a given point in time.

  • AC - Actual Cost (= Actual Cost "AC")
    The actual cost of a project corresponds to the accumulated incurred time recording, travel and material costs of all subordinate tasks at a specific point in time.

  • EV - Earned Value "EV")
    The earned value of a project corresponds to the degree of completion of all subordinate tasks at a specific point in time in relation to the planned costs and the percentage work progress determined via the target/actual comparison.

  • SV - Schedule Variance (= Schedule Variance "SV")
    The schedule variance of a project corresponds to the difference of the work value minus the planned costs of all subordinate tasks at a certain point in time.

  • CV - Cost Variance "CV")
    The cost variance of a project is equal to the difference of the activity value minus the actual costs of all subordinate tasks at a certain point in time.

  • SPI - Time Efficiency (= Schedule Performance Index "SPI")
    The time efficiency for a project indicates how large the time variance of the individual subordinate tasks is in comparison to the planned time expenditure.

  • CPI - Cost Efficiency (= Cost Performance Index "CPI")
    The cost efficiency for a project indicates how big the cost variance of the individual subordinate tasks is compared to the planned cost.



Field Descriptions

Day:

Full day of the week name.

Date:

Display of the day date for which the basic data was determined.

Plan. Eff.:

The target effort corresponds to the planned time effort in hours, man-days or man-years, etc. The estimated effort is entered when defining tasks. For projects/sub-projects, this value corresponds to the sum of all subordinate tasks.

Act. Eff.:

The actual effort corresponds to the sum of the work actually done on tasks from time, travel or material recording. For projects/sub-projects, this value corresponds to the sum of all subordinate tasks.

PAC:

Shows the percentage of a project that has already been completed. The planning effort is put in relation to the hours worked.

Any delays are taken into account in the detailed calculation.

Example:

  • without delays:
    For a task with a target effort of 100 hours, 70 hours of actual effort have already been performed => the project is 70% (=> 70 / 100) completed.
  • With delays:
    For a task with a target effort of 100 hours, 70 hours of actual effort have already been done, but there was a 5 hour delay => the project is 66% (=> 70 / 105) completed.

PC:

The planned costs (PC) are defined when a project is defined and correspond to the summed planned total internal budget (= people budget + material budget) of all subordinate tasks at a given point in time.

For the Earned Value Analysis, a linear planning progression is assumed in the specified project period, i.e. a task which, for example, goes over 10 days with a planned budget of 1,000.00 Euros is planned with a daily consumption of 100.00 Euros = 1,000.00 Euros / 10 days.
Other names for planned costs (PC):

  • Planned Value (= Planned Value "PV")
  • Target Cost of Work Scheduled "SKBA" (= Budgeted Cost of Work Scheduled "BCWS")

AC:

The Actual Costs (AC) of a project correspond to the accumulated accrued time recording, travel and material costs of all subordinate tasks at a certain point in time.
Other names for actual costs (AC):

  • Actual Value (= Actual Value "AV")
  • Actual Cost of Work Already Completed "IKAA" (= Actual Cost of Work Performed "ACWP")

EV:

The performance value (EV) of a project corresponds to the degree of completion of all subordinate tasks at a certain point in time in relation to the planned costs (PC) and the percentage work progress determined via the target/actual comparison (SI), i.e.
performance value (EV) = planned costs (PC) * target/actual comparison (SI).

The Effort Value (EV) indicates the actual value of the effort already expended at a given point in time.
Other names for performance value (EV):

  • Budgeted Cost of Work Already Completed "SKAA" (= Budgeted Cost of Work Performed "BCWP")

    • SV:

    The plan variance (SV) of a project is equal to the difference in the earned value (EV) minus the planned cost (PC) of all subordinate tasks at a given point in time, i.e.
    plan variance (SV) = earned value (EV) - planned cost (PC).

    A negative plan variance generally means, when planning is accurate, that the plan cost is higher than the value corresponding to the degree of progress already achieved, i.e. the work progress is most likely lagging behind the plan.

    A positive plan variance generally means, with exact planning, that the value corresponding to the degree of progress already achieved is higher than the plan costs, i.e. the work on the task is most likely to be completed earlier.

    CV:

    The cost variance (CV) of a project is equal to the difference of the activity value (EV) minus the actual cost (AC) of all subordinate tasks at a certain point in time, i.e.
    cost variance (CV) = activity value (EV) - actual cost (AC).

    A negative cost variance generally means, with accurate planning, that the actual cost is higher than the value corresponding to the degree of progress already achieved, i.e., the work progress is most likely lagging behind the plan.

    A positive cost variance generally means, with exact planning, that the value corresponding to the degree of progress already achieved is higher than the actual costs, i.e. the work on the task will most likely be completed earlier.

    SPI:

    The time efficiency (SPI) for a project indicates how large the time deviation of the individual subordinate tasks is from the planned time expenditure. To calculate this, the performance value (EV) is set in relation to the planned costs (PC), i.e.
    Time efficiency (SPI) = performance value (EV) / planned costs (PC).

    A time efficiency value greater than 1.00, if accurately planned, most likely indicates that tasks will be completed faster than planned, i.e., the project can be completed faster.
    For example, an SPI value of 2.00 indicates that tasks will be completed twice as fast as planned.

    A time efficiency value less than 1.00, when planned accurately, most likely indicates that the tasks will be completed slower than planned, i.e. the project is in danger of falling behind schedule or is already behind schedule.
    An SPI value of 0.50, for example, indicates that the tasks will be completed only half as fast as planned.

    CPI:

    The cost efficiency (CPI) for a project indicates how large the cost variance of the individual subordinate tasks is from the planned costs. To calculate this, the performance value (EV) is set in relation to the actual costs (AC), i.e.
    Cost efficiency (SPI) = performance value (EV) / actual costs (AC).

    A cost efficiency value greater than 1.00, if accurately planned, most likely indicates that the tasks cost less than planned, i.e., the project can be completed more cheaply.
    For example, a CPI value of 2.00 indicates that the tasks cost half as much as planned.

    A cost-effectiveness value less than 1.00, if planned accurately, most likely indicates that the tasks will cost more than planned, i.e. the project will be more expensive.
    For example, a CPI value of 0.50 indicates that the tasks will cost twice as much as planned.
