Creation > Contractors > Roles

Roles of an external contractor such as project management, project leadership, etc.


Administration of defined company roles such as "Management", "Project Management", "Administration" etc. or of freely definable project responsibility roles of external employees. The assignment of company roles has an effect on the person and project rights which this employee may see by default.




Person selection

Selection of one or more persons from a list depending on the program function.

To create or edit persons, switch to the corresponding program function under the main menu item Master data.



Field Descriptions

Person:

This selection field is used to select the person whose data is to be edited or displayed depending on the action.


Select

A click on this button confirms the selection of the person and loads the corresponding data for editing or viewing.




personal skills

Listing of an employee's skills for one or more groups.

A checkbox determines whether the employee has a skill or not.

If yes, additional information about the employee's ability can be entered in further fields, e.g. how much experience he already has in an area or how good his knowledge is in it.

In addition, internal remarks can be noted in a comment field, such as important notes or similar.

An important capability is the so-called co-worker role. By assigning these roles, the person inherits special rights, which are explained in more detail in the notes section.

NOTE:
Management
A person with the Management role has all rights to every project and document by default, and is allowed to see all information about every person except for the persons private appointments.
Project Management
Only one person with the role Project Management can be assigned to projects as project manager. This person then automatically has display rights on all sublevels of a project to which they are assigned. However, it may only make changes at the parent project level by default, unless it has been granted rights elsewhere.
Project Lead
Only a person with the role Project Lead can be assigned to projects as project lead or deputy project lead. This person then has all rights to the projects he/she is assigned to by default. In addition, this person is the only person besides the resource planner authorized to assign resources to a project..

Resource Planning
A person with the role Resource Planning has by default display rights on all projects and is allowed to assign resources besides the project manager. Consequently, this also means that they are allowed to see all people in this program function.
Administration
A person with the role Administration is allowed to see all information about each person except the personal appointments of the persons.



Field Descriptions

Del.:

Activate or deactivate an entry by placing a tick in the "checkbox".

Skill Name:

This field contains the names of the skills of the individual persons, e.g. "English", "Russian" or similar as skills for the group "foreign languages".

Exp.:

This field indicates how long an employee has possessed a particular skill, that is, how many years of experience he or she already has.

Evaluation:

The level of knowledge of a skill is to be entered in this field. The standard evaluation is based on school grades (1 = "very good", 2 = "good", ..., 6 = "insufficient").

Comment:

This field can be used to enter additional comments about an employee's ability, such as internal remarks, important notes, or the like .

Edited:

In this field, the user name is displayed or entered in text form (maximum 20 characters).

This can be structured according to the naming convention of the client and is normally managed by the administrator.

Suggestions for the naming convention: first name, last name, initials or a name chosen by the user.


Save

By clicking on this button, the changes are made. The text of the button depends on the current action mode.
