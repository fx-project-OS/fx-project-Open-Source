Reporting > Times > Performance Proof

Listing of time recording for a specific timespan with various selection criteria (project, customer, employee, etc.)


The performance record is primarily used to present the services rendered to the client within a certain period of time. A printable report with all data is created ready for sign-off via the file manager.




Set display criteria for the service confirmation.

Set display criteria for the service confirmation.

Define selection criteria according to which the performance record of the individual projects is to be prepared for the corresponding employee. In particular, the time span to be displayed can be set, as well as the underlying project.

If the display type is "Automatic", the program automatically selects the time span to be displayed.



Field Descriptions

Time Span from:

This field displays the point in time from which data for a project should be listed.

In input mode, a date can be entered here according to the user settings.

Project:

The Project field lists the tasks (projects or tasks) for which you have the appropriate rights and the corresponding filter criteria apply.

Time Span to:

This field displays the date up to which the data for a project should be listed.

In input mode, a date can be entered here according to the user settings.

Customer:

Here you can select the client of the activity (project, subproject or task).

Max. Characters:

Defines the maximum number of text characters to be displayed, i.e. if a text is too long (especially for print formatting), the text will be wrapped or truncated after the entered number of characters, depending on the "Line break" field set.

An entry of '0' means: no text wrapping. For print formatting, e.g., for the "Proof of service", the preset value of 50 (= default) is recommended.

Person:

This selection field is used to select the person whose data is to be edited or displayed depending on the action.

Max Prj. Chars:

Definition of the maximum number of text characters to be displayed for the project designation, i.e. if a text is too long (in particular for print formatting), the text is wrapped or truncated from the number of characters entered, depending on the "Line break" field set. If a value is entered, the project is also displayed without the project numbering.

An entry of '0' means: No text wrapping and project numbering (= default).

Display Type:

Specifies how certain data should be formatted within a report.

Line Break:

Specifies whether texts are to be wrapped or truncated after a certain number of characters. When text is wrapped, the text remaining after a certain number of characters is continued in the next line and possibly wrapped again, and so on.

Subtotals:

By ticking this checkbox you can define whether subtotals should be displayed or not.

Note: In day mode, subtotals are formed after each week if the checkbox is ticked.

Empty Entries:

By checking or unchecking this field, you can determine whether irrelevant entries (=blank entries) are to be displayed or not.

If, for example, a month is listed for a project, but only two days in this month were worked for this project, you can decide here whether the entire month or only these two days should be displayed.

Project No.:

This is the directory number for a project.

The directory number cannot be changed for parent projects.

The directory number is automatically adjusted when deleting or moving.

By manually changing the directory number, you can move projects within a hierarchy.

Travel Costs:

Display function only: Literal for the travel expense block in time recording.

Unknown:

If this checkbox is checked, "unknown" tasks will also be displayed.

Release:

The release indicates whether the record has been released for settlement.


Display

By clicking on this button, the list is generated and displayed. Depending on the defined selection criteria, the list is filtered accordingly.




Performance record

Performance record

The performance record for the individual employees is displayed here, depending on the time span set. The project or subproject set in the selection field serves as the base project.

All subprojects or tasks assigned to this project are listed in the report, provided that a timesheet exists for them.



Field Descriptions

Day:

Full day of the week name.

Date:

Used to enter or display a date of the day. If a date entry is required, the default value (today's date) is usually already preset.

Time:

Used to enter or display a time.

Break:

In Break the user enters the sum of his breaks taken during the activity performed.

Project:

This field displays employees who are involved in the current project.

Serv.-Type:

The activity type in cost accounting is a reference value that indicates the qualitative technical description of the production activity. Here, it can be classified between productive (=value-adding) and non-productive types.

Activity:

In this field you have the possibility to describe your work in detail. The more exact the descriptions, the more exactly the work progress can be reconstructed at a later time.

Annotation:

This field displays general comments about a project.

Editor:

Indicates which user last edited a record.

TotalHours:

This field contains the total hours for an entered activity, regardless of whether the work performed is billable or not.

Inv.Hrs.:

This field contains the billable hours for an entered activity.
