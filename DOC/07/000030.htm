Administration > Profile/User > User Definition

Create users and assign them to an already created person


The fx-project software is accessed via a user account which is protected by a password. A valid e-mail address must be assigned to the user so that he can be reached for generated mails. When defining the user, a user profile can also be assigned immediately. Furthermore, it is possible to define the data display regarding the costs. If the user is to receive administrative access rights, the corresponding check mark must be set in the assigned field. A user will always be assigned to a defined person (employee (internal / external) / client etc.).




User selection

Selection of the user to be edited.

In the User selection one selects the desired user for further processing of the data.

These can be displayed, created, changed or deleted in the User block depending on the action.

The main administrator cannot be deleted and is therefore not displayed in this mode.

NOTE:
The entries for the field "name_user" can be administrated at the menue item"Define Administration/User Administration/User".
In case you have no right on this programm resp. it is not shown in your main menue, then please contact your administrator, project manager or supervisor.



Field Descriptions

User:

Users can be selected in User.


Select

Clicking on this button will forward the selected data.




User block

Change the data of the selected user.

In User block you edit the data of the selected user.

To change the data of another user, select the user in User selection.

The main administrator cannot be deleted and is therefore not displayed in this mode.

Use Administrator-Prg. to give the user administrator rights.

New Password and Repeat New Password must match. If a password is entered it will be changed and the user will be activated if the user was inactive or locked. If no password is entered the user status will not be changed.

The user's E-Mail-Address is the address to which all outgoing emails from the program are sent. Therefore, it should be correct and up to date.

NOTE:
The entries for the field "userpersid" can be administrated at the menue item"Persons/(according Person Type)".
In case you have no right on this programm resp. it is not shown in your main menue, then please contact your administrator, project manager or supervisor.



Field Descriptions

User Name:

In User Name the account of a person is stored. This name, the password and the client ID are used to log into the system. The naming convention is freely chosen or determined by the company.

User-ID:

This User-ID is used to uniquely identify a user within a client.

E-Mail-Address:

The E-Mail-Address entered here is where all messages from the system for the user are sent. Therefore it should always be correct and up to date.

Valid from:

Date from which a record is valid.

Login:

The New Password contains the personal login password of a user. For your own security, simple passwords should not be used.
If the password is forgotten, the owner can request a new one from the administrator and change it later.

Administr.-Prg.:

Administrator-Prg. specifies whether or not the user has administrator privileges and is allowed to use its programs as well.

2FA:

Device ID for the 2FA (2-Factor-Authentication) of a user.

Valid to:

Date until when a record is valid.

Repeat Password:

To prevent typing errors, the new password must be entered once again in the field Repeat New Password.

Corresp. Person:

Select in Corresponding Person the associated person of the user to access the data (projects, ...) of the person.

Language:

The language set.

Costs:

In Costs, permissions on cost fields are given or revoked to the user. This is useful if they give the customer view permissions on project data, but the customer is not allowed to see costs.

User Prof. Grp.:

In this field you can select the desired user profile group via a drop-down menu.


Save

By clicking on this button, the changes are made. The text of the button depends on the current action mode.
