Planning > Projects > Matrix: Graphically as Gantt

Planning an existing project graphically by means of Gantt charts


Tasks (projects, sub-projects + tasks) including any existing sub-tasks can be changed or deleted quickly and easily. It is also possible to create several new projects at the same time.

The fields to be edited can be defined and selected by the user. The deadline shift can optionally also be carried out graphically.


NOTE:
It should be noted that the number of selected rows and columns can affect the amount of data and thus the response times. The maximum number of rows to be displayed can be set by the administrator (default = 100).

Project Detail
 Displays a wide variety of project attributes as a table. Project attributes are displayed as columns and are for example status, priority, effort, time period, persons responsible for the project, budget fields, descriptions or resource planning. Depending on the project type, certain contents can be changed.

The columns to be displayed can be defined in the toolbar by clicking on the icon , as well as by selecting already defined column constellations.

Project Attributes Columns:
StatusSelect box: Scheduled, Active, Inactive, or Completed
PrioritySelect box: low, medium, high, or immediate
Planned EffortInput field: Planned target effort in hours, which can be defined for tasks and is cumulated to higher-level projects.
Actual EffortOnly display field: Actual effort in hours. Values are determined from the time recorded for tasks and are cumulated to higher-level projects.
Max. EffortInput field: Maximum actual effort in hours, which can be defined for tasks and is cumulated up to higher-level projects. This value defines the upper limit of billable actual hours, i.e. up to this value times may be recorded.
Planned Period of TimeDisplay field only: Planned target period. These values are determined from start and end times of the respective operation in the Gantt.
Actual Period of TimeDisplay field only: Actual period. These values are determined from start and end times from completed time recording for tasks and are cumulated to higher-level projects.
DeadlineInput field: Planned final date by which a task should be completed. If a task is moved beyond the deadline, the deadline will be adjusted automatically.
Project ManagerSelection field: Main responsible and mostly administrative person on main project level. Supervisor and contact person of project managers from subordinate subprojects/tasks. The project manager can only be set at project level and inherits automatically downwards.
1st Project LeaderSelect field: main responsible and mostly administrative person at subproject/task level. The project manager is mostly responsible for resource planning, managing and controlling of his assigned subprojects/tasks.
2nd Project LeaderSelection field: The deputy project manager has in principle the same area of responsibility as the project manager. However, he usually only appears when the project manager is out of the house.
Customer ContactSelection field: Defines the contact person or the client of a project, i.e. who has assigned the project. This value is especially important for invoicing, since invoices for this task can only be issued to the person specified here.
Accounting Approach (Persons)Select field: Fixed price, Per Hour Basis or Not Invoiceable. For fixed price tasks, up to 100% of the planned budget can be invoiced from active tasks on a percentage basis (Time registration is not required). For per hour based tasks, the amount to be invoiced is determined by the registered times. For non-invoiceable tasks, only travel costs can be accounted for and not hourly costs.
Accounting Approach (Material)Select field: Fixed price, Per Hour or Unit Basis or Not Invoiceable. For fixed price tasks, up to 100% of the planned budget can be invoiced from active tasks on a percentage basis (Material registration is not required). For per hour or unit based tasks, the amount to be invoiced is determined by the registered materials. For non-invoiceable tasks no material can be invoiced.
Budget TypeSelect field: Informative field to distinguish budgets. The values listed here come from the self-definable lookup values.
Int. HRB FrameInput field: the "Internal person budget frame" can only be entered for projects. It sets the estimated upper limit of all expenses for persons in all subordinate tasks.
Int. Plan. HRBInput field: The "Internal planned person budget" can only be entered for tasks. It specifies the estimated upper limit of the incurred expenses for persons.
Int. Used HRBDisplay field only: The "Internally planned person budget" shows the incurred expenses for persons. This value is determined from the time recording of each employee (= hours * purchase price).
Ext. HRB FrameInput field: The "External person budget frame" can only be entered for projects. It sets the estimated upper limit of all summed up revenues of persons in all subordinated tasks.
Ext. Plan. HRBInput field: The "External planned person budget" can only be entered for tasks. It sets the estimated upper limit of the summed up revenues of persons.
Ext. Used HRBDisplay field only: The "Externally planned person budget" shows the summed up revenues of persons. This value is determined from each employee's time tracking (= hours * sales price).
Ext. Inv. HRBDisplay field only: The "Externally billed person budget" shows the summed billed revenue of persons. This value is determined from the accounting of fixed price person revenues, hourly base person revenues and travel expenses.
Int. MRB FrameInput field: The "Internal Material Budget Frame" can only be entered for projects. It specifies the estimated upper limit of all incurred expenses for material in all subordinate tasks.
Int. Plan. MRBInput field: The "Internally planned material budget" can only be entered for tasks. It sets the estimated upper limit of the incurred expenses for material.
Int. Used MRBDisplay field only: The "Internally consumed material budget" shows the incurred expenses for material. This value is determined from the material allocation.
Ext. MRB FrameInput field: The "External material budget frame" can only be entered for projects. It sets the estimated upper limit of all summed up receipts of sold materials in all subordinated tasks.
Ext. Plan. MRBInput field: The "External planned material budget" can only be entered for tasks. It sets the estimated upper limit of the summed revenues of sold materials.
Ext. Used MRBDisplay field only: The "Externally consumed material budget" shows the summed revenues of sold materials. This value is determined from the material allocation.
Ext. Inv. MRBDisplay field only: The "Externally billed material budget" displays the totaled billed revenue of sold materials. This value is determined from the billing of fixed price material revenue, hourly base material revenue and individual material revenue.
Project CategoryDisplay Field: Informative field to distinguish and group projects. The values listed here come from the user-definable lookup values.
Skill GroupSelection field: Specifies the most needed skill for a successful completion of this project - this value serves e.g. as a filter criterion when assigning resources or for reports. The values listed here come from the self-definable lookup values.
Method of Progress Det.Select field: defines how the progress of this project should be determined. Possible values: comparison target/actual effort (= default), completion percentage, method 0/100, method 0/50/100, remaining effort in hours + remaining effort percentage.
Reference NumberInput field: Informative field to distinguish projects.
DescriptionInput field: Detailed text to describe the project.
AnnotationInput field: Text for project-specific features.
Internal AnnotationInput field: Text for company-internal project-specific features.
Assigned HRsSelect field: Specifies the resource assignment, i.e. who should process this task. For tasks, only a direct 1:1 assignment is possible, i.e. one person is responsible for one task. If more than one person should work on this task, the extended resource planning area or the independent resource planning program functionality must be used.
Purchase PriceInput field: Sets the purchase price (= EK) for the assigned person. The EC is the basis for the determination of the internal person budget, i.e. how much costs the employee of the company per hour of work on this task.
Selling PriceInput field: Sets the sales price (= VK) for the assigned person. The VK is the basis for the determination of the external person budget (= billable amount), i.e. how much revenue the employee brings to the company per hour of work on this task.
Gantt
 The Gantt area consists of a table-like structure with date fields as columns and projects as rows. The boundaries of the main project are represented by dashed vertical lines. The headline has the following action icons.

Action Icons:
|<Scroll to the top of the time range.
>|Scroll to the end of the time range.
+ ++Minimally or maximally increase the time range, i.e. add some or more additional date columns either at the beginning or at the end. The icons will be displayed if scrolled to the time range start or end.

At the lower left edge there is a fade-in navigation overview to quickly scroll to the desired time range. The navigation overview can be expanded and collapsed by clicking on the arrow icon.

Task:

A task consists of a graphical bar, as well as a start and end node. Operations are graphically distinguished as follows:

Project/Subproject (= collective task)Used for grouping tasks.
Task (= work package)Used for recording work steps. Only to tasks efforts can be planned, resources assigned and times recorded etc.

An operation can have different statuses, which are also represented graphically in a task:

planned (= light grey + rounded)The task is still in the planning phase, i.e. the effort and budget estimation has not yet been completed. No actual data can be collected yet.
active (= dark grey + rounded)The task is active, i.e. the planning phase is completed and the task can be worked on. Time, material and travel costs can be recorded.
inactive (= light grey + angular)The task is inactive, d.i.e. the planning phase has been completed, but no time, material or travel costs should be able to be recorded for it.
completed (= black + square)The task is completed, i.e. the task goal has been achieved and no additional time, material or travel costs should be able to be recorded.

Assigned resources of a task are displayed as an attached text box (= small flag) to the right of the task. If there is more than one assigned resource, the number is displayed in bold.
A subordinate hierarchy of a project/subproject can be expanded by clicking on the icon or expanded again by clicking on the icon.
A task can be renamed by double-clicking on a task name in the bar or by pressing [F2] on a selected task.
By clicking with the left mouse button on a task bar, this task will be marked, and the following action icons will be shown:

Action icons:
Copy highlighted operation including all subordinate operations graphically.
Move selected process including all subordinate processes graphically.
Graphically move selected operation including all subordinate operations vertically.
Expand or shrink resource planning area.
Delete process incl. all subordinate processes.

By clicking and holding the left mouse button on a process bar, this process can be shifted in time. Please note that a subproject or a task cannot be moved beyond the time frame of the parent project.
By clicking and holding the left mouse button on a start or end node, this task can be extended or shortened. Note here that a parent project cannot be made shorter than its child predecessors.

Milestone:

There are two different types of milestones:

DynamicA dynamic milestone is connected to the start or end node of a task, a time shift of x days is possible. A dynamic milestone automatically moves with the assigned task.
FixedA fixed milestone is connected to the task, but it is independent of time. A fixed milestone always remains fixed at its position and does not move with the assigned task.

By double-clicking on a milestone name, or by pressing [F2] on a selected milestone, it can be renamed.
By clicking and holding down the left mouse button on a milestone, it can be moved in time.

Dependencies:

Dependencies are represented by arrows from a node of one task to a node of another task. There are four different types of dependencies:

(ES) End-to-Start
(EE) End-to-End
(SS) Start-to-Start
(SE) Start-to-End

A dependency can be created graphically by clicking and holding down the left mouse button on a chaining symbol and then dragging the dependency arrow to the desired start or end node of the target operation. Invalid dependency relationships are shown as "red" arrows .

Resource Planning:

The resource planning section is again divided into two vertical sections.


Resource Planning
 Displays assigned and selected resources, sorted in descending order by their name.
A resource can be assigned to a task by checking the checkbox in front of it, or removed by unchecking it. A resource assigned to the task is displayed green.
A resource can be removed from a task by clicking on the delete icon to the right of the resource name and simultaneously removed from the view. The resource will be written back to the resource pool.
Clicking on the resource pool icon at the end of the resource listing will open the resource pool.

Resource Pool:
NameName, first name (username) and [employee type] of the resource. By clicking on the add icon at the beginning of the line, the corresponding resource is added to the resource list. The resource is not assigned to the task.
Available hours.Available hours of the corresponding resource in the selected period. The hours are calculated from the target hours of the corresponding resource in the selected period minus already assigned active planned hours of tasks, which are not displayed in the Gantt, i.e. which do not belong to the current parent project.
Standard PPPurchase price (= PP), which was stored for the corresponding resource. Resource was deposited.
Standard SPSelling price (= SP), which was deposited for the corresponding resource.
SkillsSkill groups und skills that were assigned to the corresponding resource. A matching skill with the skill stored in the task is highlighted in red.

The individual resource attributes are displayed as columns. Depending on the assignment, certain content (percentage, purchase price, or sales price) can be changed.

Resource attribute columns:
PercentageInput field if more than one resource is assigned, otherwise Only display field: Values can be entered from 1.00% to 99.00%. For a single assigned resource the value always corresponds to 100.00%, for an unassigned resource 0.00%.
Purchase priceInput field, if the resource is assigned, otherwise Only display field: Purchase price (= EK), which should be used for the corresponding resource specifically for this task. This value is preset with the default price of the corresponding resource.
Selling priceInput field, if the resource is assigned, otherwise Only display field: Sales price (= VK), which should be used for the corresponding resource specifically for this task. This value is preset with the default sales price of the corresponding resource.
Entry dateDisplay field only: Date when the corresponding resource joined the company (or when the resource joined the company). Resource has joined (or will join) the company.
Exit dateDisplay field only: Date when the corresponding resource has left (or will leave) the company. Open, if the date has not been set yet.
Working daysOnly display field: Number of working days per week, as well as the corresponding weekday count.
Working hours/weekOnly display field: Number of target working hours per week (according to the employment contract).
Working hours./dayDisplay field only: number of target working hours per week (according to the employment contract) divided by the number of working days per week.
