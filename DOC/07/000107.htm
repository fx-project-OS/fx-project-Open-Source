Search and list certain documents in the Document-Management-System (DMS)


Search for documents in the DMS (Document Management System) based on their maintenance structure. The following search criteria can be entered: "Category", "Text", "Period" or "File size".

Only results are listed to which you have the corresponding rights.

Existing documents

Existing documents

The results returned by this search are displayed in the form of a list. The text passages that match the search text are highlighted.

Documents for which you do not have access authorization are not displayed. The information about how many documents were found and how many are displayed can be found in the information line. Clicking on one of the icons in the action line executes the desired action:

Display: Display this file in a new browser window.

Download: Download selected file(s).

When the document is created, all rights for this document (only for the creating user, business manager and system administrator) are automatically assigned. To make the entered document accessible to other users, click on this icon.

If icons are grayed out, the logged-in user does not have the necessary access rights for this function.

In addition to the two search masks (structure and text search) for documents, there is another option to display a category with assigned documents:

In the menu bar there is a button (with an empty sheet and a pair of glasses placed in front of it), with the help of which you can display all documents that have been assigned to the called program function.

In other words, when you call certain program functions (employee, customer, project, etc.), the category of the document parent object (an object to which you assign documents) is determined.

The icon is deactivated by default and is only activated when, on the one hand, a suitable program function has been called and, on the other hand, documents have also already been assigned to objects of this category.

NOTE:
Example:
A document has been created for employee XYZ. By calling the program function "int. Employee data" the icon shown above is activated, because (at least) one document has already been assigned to an employee. If you click on it now, a list with the documents assigned to the employee appears (if you have sufficient access rights).
The functionality is essentially the same as the structured document search, except that the category is just determined with the called program function. This is especially useful if the desired program function is already active.



Field Descriptions

Chg.:

This field indicates the time of the last change in TimeDate format (YYYYMMDDHHMMSS).

Each time the record is written to, a new time stamp is stored in conjunction with the user ID.

Through this mechanism, both security and data integrity aspects are well covered.

File Name:

The name of the generated file must be entered in this field.

Auto:

If this field is activated, a new version is automatically created when changes are saved. This has the advantage that even the smallest changes are not lost.

The disadvantage is that the number of available versions quickly becomes confusing when there are many small changes.

Version:

The version number is incremented by 0.001 when a changed document is saved normally, and by 1 when a version change is explicitly requested or when the floating point range overflows (for 999 changes without a version change).

Manually changed versions are saved as a new file and activated, the old version is marked as inactive.

Size:

This field indicates the physical size of the file. With this information you can estimate, for example, how long a download would take with the usual bandwidth (-> Internet access).

Category:

In this field, the category is selected via a selection field.

Description:

Explanatory descriptions, details, comments or similar.

Changes:

The changes made must be entered in this field (in bullet points).

Key Words:

This field contains the keywords that appear in the text of the document.

Filing Place:

Specifies the storage location of the original document.

GIF-Icon:

Specifies which icon is used with this entry.
