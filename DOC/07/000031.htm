Planning > Projects > Search

Search and list specific projects, subprojects or tasks


Extensive selection and filter criteria such as search text, time period, project name, PCA code, project owner, etc. can be used to search for specific projects. All filter criteria can be combined with each other to sift the number of hits until the desired result is displayed.




Project search

Project search

Search for a specific project.

The criteria that can be entered are:
- "Project name"
- "project type= type of project"
- "planned start date"
- "planned end date".
If no criterion is entered, then all existing projects are listed.



Field Descriptions

Search Text:

This field is used to enter the string to search for.

Search from:

In this search field you can define the validity start of the time frame of the search.

To:

In this search field you can set the validity end of the time frame of the search.

Project Type:

In this field the project type is displayed or can be chosen from a selection list, i.e. this field indicates which project it is, whether "mother project", "sub-project" or "task".

Name of Project:

Display of the project name or the name of the activity (project, subproject, task).

1st Project L.:

In this field the manager of a project is displayed or must be selected from a selection list. The project manager is defined once in the master project, and is then preselected as a PROPOSAL in all underlying projects.

Green displayed persons in the selection field are project leaders, which are already assigned to at least one project in this role.

Project managers are created using the role mask with role "Project Manager".

PCA Code:

PCA is short for Pproject Codification Algorithm, which is a unique identifier to each project.

Customer:

Here you can select the client of the activity (project, subproject or task).

ID:

Primary key of the entry.


Search

Clicking on this button starts the search.




Overview of all found projects

Overview of all found projects

Here you get an overview of all found projects of the previous search.

The adjacent button for each project found takes you to the project input mask for this project.



Field Descriptions

No.:

This is the directory number for a project.

The directory number cannot be changed for parent projects.

The directory number is automatically adjusted when deleting or moving.

By manually changing the directory number, you can move projects within a hierarchy.

Project Name:

Display of the project name or the name of the activity (project, subproject, task).

ID:

Primary key of the entry.

Project Type:

In this field the project type is displayed or can be chosen from a selection list, i.e. this field indicates which project it is, whether "mother project", "sub-project" or "task".

Pln. Start:

The target start date of the project is displayed or specified here, i.e. from when the project is to be started according to planning.

Pln. End:

The target end date of the respective reference object, e.g. project, time data, trip data, etc. is displayed or specified here, i.e. by when the object should be completed according to planning.

1st Project L.:

In this field the manager of a project is displayed or must be selected from a selection list. The project manager is defined once in the master project, and is then preselected as a PROPOSAL in all underlying projects.

Green displayed persons in the selection field are project leaders, which are already assigned to at least one project in this role.

Project managers are created using the role mask with role "Project Manager".

Customer:

Here you can select the client of the activity (project, subproject or task).

PCA Code:

PCA is short for Pproject Codification Algorithm, which is a unique identifier to each project.

Link:

This field is optional. If a link is entered, the automatic user rights query is bypassed and the Trans_ID field is cleared.
