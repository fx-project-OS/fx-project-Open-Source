Reporting > Projects > Net Plan Diagram

View of a project as net plan diagram with critical path


Structured graphical display of an entire project hierarchy, with each project level displayed in its own column.

Based on the project duration, the earliest start and the earliest end are calculated in forward planning, as well as the latest start and the latest end in backward planning, if a global end date has been specified.




Determine selection criteria

Define selection criteria for displaying the network.



Field Descriptions

Main Project:

Top level of a project structure, under which all subprojects and tasks are subordinated.

Pln. Start Date:

The target start date of the project is displayed or specified here, i.e. from when the project is to be started according to planning.

Critical:

If this checkbox is selected, then the critical path is also determined and displayed with red lines. The number of buffer days between two consecutive tasks serves as a basis.

Customer:

Here you can select the sold-to party of the task (project, subproject or task).

Pln. End Date:

The target end date of the respective reference object, e.g. project, time data, trip data, etc. is displayed or specified here, i.e. by when the object should be completed according to planning.

Align Workd.:

Align time dates with working days, i.e. a date must not fall on a weekend or a public holiday.

Project Manag.:

Here the project manager of the task (project, subproject or task) is selected.

Buffer Day(s):

Buffer day(s) = minimum interval in days between two consecutive tasks.

1st Project L.:

Here the project manager of the activity (project, subproject or task) is selected.


Display

By clicking on this button, the list is generated and displayed. Depending on the defined selection criteria, the list is filtered accordingly.




Display

Determination of the start and end dates of a complete project structure by means of project duration of the individual tasks (forward and backward planning).


Forward planning:
FB (Earliest Start):
= the earliest end of its predecessor plus number of given buffer days between two tasks. Possibly alignment with working days.
FE (Earliest End):
= the earliest start of this task plus number of duration in days. Possibly alignment to working days.


Backward scheduling:.
SE (Latest End):
= the latest start of its successor minus a buffer day. Possibly alignment with working days.
SB (Latest Start):
= the latest end of this task minus the duration in days. Possibly alignment to working days.


Buffer days:.
GP (total buffer):
= Day difference between latest start and earliest start.
FP (Free Buffer):
= number of days between earliest start of successor and latest end of this task.
