Define warnings

Defining warning messages or direct output of problem projects
A certain number of standard warning messages are available to you, which you can link to a specific percentage. These details are either displayed only or saved to the database.
Save data when button Save is clicked: the data you select is saved to the database and is available to the warning message program (which runs during the night). This evaluates the data and sends an email to the relevant project manager containing all the projects (as well as key project metrics) that have triggered an appropriate warning message.
Show data when button Display is clicked: the data you selected will be evaluated immediately and a report will be issued containing the projects that triggered a warning message. Caution. To save this warning message configuration, you must press the Save button, only then will the data be saved.
If the field Displ. Sub Projects is selected, the subprojects are also included in the reporting, this applies to the output in the table as well as to the automatic check if the data was written to the database.


NOTE:
Warning messages assigned to the value "inactive" will not be included in the report nor in the e-mail warning.

Field Descriptions

Proj. No.:

This is the directory number for a project.

The directory number cannot be changed for parent projects.

The directory number is automatically adjusted when deleting or moving.

By manually changing the directory number, you can move projects within a hierarchy.

Project Name:

In the project name, the name or designation of the task (project, subproject, task) is specified.

Planned Effort:

The target effort corresponds to the planned time effort in hours, man-days or man-years, etc. The estimated effort is entered when defining tasks. For projects/sub-projects, this value corresponds to the sum of all subordinate tasks.

Actual Effort:

The actual effort corresponds to the sum of the work actually done on tasks from time, travel or material recording. For projects/sub-projects, this value corresponds to the sum of all subordinate tasks.

Pln. Start:

The target start date of the project is displayed or specified here, i.e. from when the project is to be started according to planning.

Pln. End:

The target end date of the respective reference object, e.g. project, time data, trip data, etc. is displayed or specified here, i.e. by when the object should be completed according to planning.

Int. Plan. Bud.:

Indicates the internal planned budget. For tasks this value is to be entered as an estimated value, for (sub-)projects this is the sum of the individual values of the tasks below.

For example, if you enter 3500 € in task A, and 2500 € in task B, and both tasks are under subproject T, the amount of the budget in subproject T is set to 6000 €, generally: A + B = T.

Int. Spent Bud.:

This value is calculated for tasks from the number of hours entered in the time recording * hourly rate EK, for (sub-)projects from the sum of the individual used budgets, of the underlying tasks, which are placed under the (sub-)projects in each case.

For example, if you have used €3000 in task A and €500 in task B, and both tasks are under sub-project T, the amount of the budget used in sub-project T is set to €3500.
generally: A + B = T

Deadline:

Specifies the deadline (latest time) by which the project must be completed.
