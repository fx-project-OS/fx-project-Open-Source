Reporting > Progress > Project Progress Estimation

Enterability of a progress estimation for assigned tasks depending on the progress method


Possibility to enter a progress estimate for assigned tasks. This ensures that the degree of completion is available as accurately as possible at all times.




Selection of tasks

Selection of tasks

Select the task for which you want to estimate the cumulative quantity.

If you select a parent project or subproject, all subordinate tasks of this project are listed.

If you select [All], all tasks are listed.



Field Descriptions

Tasks:

Here you can define whether tasks (and possibly to-dos) should also be displayed in the project selection list or not. If this checkbox is not set, only projects and subprojects are displayed.

Checkbox:

Activation of the checkbox hides tasks for which no estimation can be made

Checkbox:

By activating the checkbox not own tasks of resources are shown, to which one is assigned as project manager

Checkbox:

By activating the checkbox only tasks of still "open" requests will be displayed.

Checkbox:

By activating the checkbox, only tasks are displayed, for which a temporal delivery of an estimate is required


Display

By clicking on this button, the list is generated and displayed. Depending on the defined selection criteria, the list is filtered accordingly.




Enter progress estimate

Enter progress estimate

Depending on the selection made, the corresponding active tasks are listed here. At the top, those to which you are assigned as a resource. In the next position, those to which no resources are assigned and to which you are the project manager.

For each task, you can now enter an estimated value and an optional comment, depending on the defined method.



Field Descriptions

Project Name:

In the project name, the name or designation of the task (project, subproject, task) is specified.

Method:

Determination of the method how the project progress (= percentage of completion) is to be determined.

No.:

The percentage of the estimate is selected here.

Text:

Display of the unit (percent)

Annotation:

This field displays general comments about a project.

Statu:

Indicates the (completion) status of a ToDo item.

Changed:

This field indicates the time of the last change in TimeDate format (YYYYMMDDHHMMSS).

Each time the record is written to, a new time stamp is stored in conjunction with the user ID.

Through this mechanism, both security and data integrity aspects are well covered.

Time Per.:

Displays the runtime of the project in a combined field.

Priority:

Indicates the urgency of the (sub)project.
The higher the priority of a project, the higher it is displayed in the control panel of an employee.

Link:

This field is optional. If a link is entered, the automatic user rights query is bypassed and the Trans_ID field is cleared.
