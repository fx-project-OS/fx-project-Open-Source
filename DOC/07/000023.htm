Plan an existing project in text form


In the core of fx-project tasks (projects, sub-projects and tasks) can be managed, i.e. recorded, modified, deleted or displayed.

The data areas thereby extend into six data blocks, namely:

  • Data block (project number, name, status, main, additional and executing persons, description etc.).)
  • Date block (Start date, End date, Deadline, Target effort etc.)
  • Finance block (Budget frame, Settlement type, Internal and External budget fields etc.)
  • Progress block (Determination method + reporting dates)
  • Cost master data block (Cost objects, cost centers + cost elements)
  • Info block (Reference number, notes, etc.)

Data block for the activity (project, subproject, task)

Information on current projects

You will see various fields that contain general information about your project, such as project name, project manager, customer, etc.

Furthermore, a little further down, three history fields are displayed that reflect the state of your project.

Depending on how the individual bars are colored, so is the state of the project:

- green = within planning,

- yellow = slightly exceeded [0%-10%],

- red = serious exceedance [+10%].

The filling level indicates how many of the individual measured values have already been required.

Example: A half-filled red bar means that 50% of the time, effort, or budget has been used so far, but the project itself is not yet 50% complete (but at most 40%, since the bar is colored red). If the bar were yellow, the degree of completion would be between 41% and 50%, if it were green, the degree of completion would be greater than 50%.
=> The fill level indicates the percentage used value of the field, the color signals how this value relates to the POC.

If you move the mouse over a bar, a window is displayed in which you can see the graphically visualized information again in plain text.

The project manager and the deputy are given access to the project, as are any employee resources that may be assigned later (see Resource Planning).

If you enter a skill here, this will later be taken into account in resource planning, i.e. only those employees will be proposed who have exactly this skill.

If the status of a (sub)project is changed, this change also affects all subordinate projects.

Literal for the progression field of time, occurs in conjunction with the progression symbol, which, depending on the coloring, visualizes the progression or state of time.

--> ( budget-consumed / budget ) * 100% = percentage of cost expenditure

Progress Block

Definition of the project progress determination
At each project level, you can define how the project progress (=completion rate) should be determined, but only for tasks, progress can be reported depending on the reporting time set.
The following methods are available:

  • Comparison of Target/Actual Effort
    The degree of completion is calculated from the sum of the actually performed actual effort for the respective tasks divided by the sum of the planned target effort plus the sum of the planned additional effort.
    This method is the most accurate, but requires a well-planned and carefully considered target effort.
    If the sum of the target efforts equals 0, i.e. no effort has been planned for any task, then the percentage of completion is always 100%.

  • Remaining effort in hours
    If an effort has already been made, then the percentage of completion is calculated by dividing the sum of the efforts made for the respective tasks by the sum of the efforts made plus the estimated remaining effort in hours.
    If remaining effort is estimated equal to 0 this means that there is no more work to be done, the task is complete and therefore the percentage of completion is equal to 100%.

  • Method 0/50/100
    The degree of completion results from the value, thereby means:
    • 0: The task has not yet been started.
    • 50: The task is being worked on.
    • 100: The task has been completed.

  • Method 0/100
    The degree of completion results from the value, thereby means:
    • 0: The task has not yet been started or the task is being worked on.
    • 100: The task has been completed.

  • Percentage remaining
    The percentage complete is obtained by subtracting 100 with the estimated percentage remaining.

  • Percentage complete
    The percentage complete is equal to the estimated percentage.

Adapt

If the time frame of a project exceeds that of its parent(s), the "Adjust" button can be pressed, which will adjust the time frames to the new maximums.

Example: Project 2 is to be created under project 1. Project 1 starts on 3/1/2002 and ends on 4/25/2002, but project 2 should already start on 2/28/2002. To save you some steps, you can click the "Adjust" button after the system has recognized the conflict. This will set the start of project 1 to 2/28/2002, and the same will happen for any parent projects of project 1.




Resource assignments

Overview of the resources assigned to a project, subproject and/or task.



Field Descriptions

Name:

In this field the family name of the person or the name of the legal entity (e.g. company name) is displayed or entered.

Material Name:

Name or description of a material.

Material Type:

Type of a material, that is, to which material group it belongs. The material types can be defined as lookup values themselves.

Group:

This field displays the individual employee skill groups, that is, the employee areas that they cover.

Skill Name:

This field contains the names of the skills of the individual persons, e.g. "English", "Russian" or similar as skills for the group "foreign languages".

Start:

This field specifies the plan start date. When created, it is identical to the project start date when the employee was assigned, but it can be changed later.

End:

This field specifies the plan end date. When created, it is identical to the project end date on which the employee was assigned, but it can be changed later.

PP:

Monetary value, the amount of which shows the financial expenditure for the company per hour of use of a worker or a material (counterpart to VK=sales price). Here the EK (purchase price) is to be entered, which the resource receives for the current project.

Example: An employee has a purchase price of 25 €. If he works on a "normal" project, his purchase price is also 25 €. But if he works on a project that needs all his experience, the price can be adjusted to this project (e.g. price 40 €). The adjustment is also possible for the sales price.

SP:

Monetary value, the amount of which shows the financial return for the company per day of use of a worker or a material (counterpart to EK=purchase price). The sales price that the resource generates for the current project must be entered here.

Example: An employee has a sales price of 40 €. If he works on a "normal" project, his sales price is also 40 €, but if he works on a project that requires all his experience, the sales price can be adjusted to this project (e.g. sales price 60 €). The adjustment is also possible for the EK.

Effort:

Specifies how many hours an employee should spend on a task.

%:

Indicates the percentage of work performed by the employee.

Utilizat.:

Indicates the percentage to which a resource is already utilized (by other projects).

Materials always have 100% capacity utilization when allocated.

For employees, the utilization is calculated from the values of the target workday and the target effort for the task.
For example, an employee with a standard eight-hour day is assigned to a task with a target effort of 24 hours. The duration of the task is three days.
=>Within these three days, the employee can work three times eight hours according to the target workday
=>24 hours / (3 * 8 hours) => 1 => Utilization of the employee is exactly 100% for this task.
Now let's assume the same task is now to be completed in four days
=>24 hours / (4 * 8 hours) => 0.75 => Utilization of the employee is exactly 75% for this task.
If more than one employee is assigned to a task, the workload can be split so that, for example, one employee does 16 hours and the other does 8 hours. With a standard eight-hour day and a task runtime of four days, the workloads of the two employees in the case just mentioned would be 50% (=> 32 / 16) and 25% (=> 32 / 8) respectively.

The total workload of an employee for an appointment is calculated from the individual workloads within the tasks for this appointment, i.e. if an employee has three tasks assigned on one day, with individual workloads of 40%, 30% and 20%, then the total workload for this employee for this day would be 90%.

Missing days and holidays are taken into account in the workload calculation
