Administration > Access Rights > Overview of Rights

Overview of assigned access rights


All access rights of projects, documents and reports of a single person can be displayed.




Select user and reference

Select user and reference

Select reference and user for which the existing rights are displayed.

For projects, the project tree is displayed with authorized projects and paths that lead to authorized projects.

In any case, only projects from the project menu are displayed (only the projects that the logged-in user is allowed to see and the same criteria that are selected in the project menu apply).

The projects to which the selected user has rights are highlighted with darker background.



Field Descriptions

User:

This field contains an internal key for each user entry (user ID).

Reference:

Selection of the reference for which the user has been assigned the rights.


Accept

Clicking this button saves the entered data and continues the program execution.




User has explicitly assigned rights for the following references

A list of projects, reports, documents, etc. for which the user has authorization is displayed.

List is displayed for all selected users with authorization.

This list includes: Projects, Reports, Documents, etc. (depending on the selected reference).

The access rights for documents are assigned to individual actions.



Field Descriptions

Description:

Designation of the data type.

Disp.:

This field specifies whether the related field or program function may be displayed for this profile.

Crea.:

This field specifies whether the relevant field may be created for this profile.

Edit:

Here you specify whether the field for this profile may be changed.

Del.:

This field indicates whether the related field for this profile may be deleted.
