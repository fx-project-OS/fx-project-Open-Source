Execution > Travels > Approval

Approval of the recorded travels, expenses, daily allowances or receipts by employees, contractors or superiors


Release of travel expense entries in the relevant control instance. I.e. before a possible settlement an entry should run through the release procedure. This ensures that correct data is available.

The control instances are arranged in the following sequence:
(1) MA=employee - (2) PL=project manager - (3) VW/GL=administration/management.




Selection criteria

Restricting the display quantity

In Selection criteria the period and project branch are restricted. This allows for individual, more manageable displays.



Field Descriptions

Time Span from:

This field displays the point in time from which data for a project should be listed.

In input mode, a date can be entered here according to the user settings.

Text:

Text field

Variable text field that can be customized programmatically.

Time Span to:

This field displays the date up to which the data for a project should be listed.

In input mode, a date can be entered here according to the user settings.

Ignore Release:

Ignore release

Project:

The Project field lists the tasks (projects or tasks) for which you have the appropriate rights and the corresponding filter criteria apply.

Max. Project Chars:

Definition of the maximum number of text characters to be displayed for the project designation, i.e. if a text is too long (in particular for print formatting), the text is wrapped or truncated from the number of characters entered, depending on the "Line break" field set. If a value is entered, the project is also displayed without the project numbering.

An entry of '0' means: No text wrapping and project numbering (= default).

Person:

This selection field is used to select the person whose data is to be edited or displayed depending on the action.

Max. Text Chars:

Defines the maximum number of text characters to be displayed, i.e. if a text is too long (especially for print formatting), the text will be wrapped or truncated after the entered number of characters, depending on the "Line break" field set.

An entry of '0' means: no text wrapping. For print formatting, e.g., for the "Proof of service", the preset value of 50 (= default) is recommended.

Customer:

Here you can select the client of the activity (project, subproject or task).

Line Break:

Specifies whether texts are to be wrapped or truncated after a certain number of characters. When text is wrapped, the text remaining after a certain number of characters is continued in the next line and possibly wrapped again, and so on.

Time Data:

In this field the user sees the start and end times of the activities performed.

Travel Costs:

Here you can select how the travel expenses entered in the time recording are to be included in the listed calculation.

Possible settings:


Display travel expenses in separate row:
- Travel expenses are included and displayed on the next line of the project to which they are assigned.

Add travel expenses to employee costs:
- Travel expenses are included and added to employee costs in the same row of the project to which they are assigned.

Show Travel Expenses Alone:
- Only all travel expenses incurred alone will be displayed in the listing, meaning that all other employee hourly costs will not be included in the calculation.

Do not add travel expenses:
- Travel costs are not included in the calculation, only employee hourly costs are listed.

Material:

If this checkbox is set, materials are displayed.

Annotation:

This field displays general comments about a project.


Display

By clicking on this button, the list is generated and displayed. Depending on the defined selection criteria, the list is filtered accordingly.




Share

List of recorded times, travel expenses or material consumption depending on defined selection criteria. The individual entries can then be released. Released entries can no longer be changed by a lower instance. If authorized, the release can also be reversed.
