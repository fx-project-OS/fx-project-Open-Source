Manage project categories

Manage project category groups and project categories.

With a click on the "Create new project category group" icon in the lower left corner of the "Prj. category group" column, a new group is dynamically integrated into the mask using JavaScript. Each group consists of a changeable sort number, as well as a changeable name and any number (or even none) of subordinate project categories.

By clicking on the "Add" icon in the "Project categories" column, any number of categories can be dynamically created for a group. A category also consists of a changeable sort number, as well as a changeable name. These categories can be assigned to the projects in the individual project program functions.

By clicking on the "Delete" icon to the right of each group or category field, they can be deleted. If a category is already assigned to at least one project or if the group or category is a default group or category of fx-project, then this icon is inactive and the category cannot be deleted. If a group is deleted which contains one or more non-deletable categories, then these categories are moved to the group "Unassigned" before the selected group is deleted.

By clicking on the "Move" icon on the left in front of each category line, the assignment to the individual groups can be changed. If only one other group exists, then the category is moved immediately, otherwise the target group must be selected in a popup window.

Any new entries, changes or deletions made are not applied in fx-project until the [Save] button is clicked.
