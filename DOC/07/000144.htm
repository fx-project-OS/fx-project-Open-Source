Import projects from a MS-Project® CSV file into fx-project®


Reads in a CSV file created with MS-Project® and creates a new project structure in fx-project from it. This makes it very easy to exchange data between fx-project and MS-Project®.


NOTE:
The fields used in the CSV file can be freely defined and adapted.




Import projects from MS-Project

Here you can import existing projects from MS-Project® to fx-project.

The project in MS-Project® must be saved as a text file (separated by tabs).

The following fields are necessary (on the left name in MS-Project®, in brackets the name under which the field must be exported in order to be recognised by fx-project):
- Outline number (position)
- Name (name/project description)
- Start (start/target_start)
- End (End/Target_End)
- Work (Work/Target_Effort)
- Predecessor (Predecessor/Dependent_of)

Optional fields:
- Created (Created/Date Created)
- Notes (Notes/Description)
- Priority (Priority)
- %Completed (Percent_Completed/Percentage_of_Completion => then sets project status to Planned or Active)

Select a valid (text) file and fill in the following selection fields with the respective people you want.

Furthermore, specify in which unit the work was exported from MS-Project® (minutes, hours or similar) and in which format the date is.

NOTE:
None of the dates to be imported may contain text characters.



Field Descriptions

GIF-Picture:

Specifies which icon is used with this entry.

Document:

Document file

Selection of the activity (project, subproject, task) for which you are authoriz:

The Project field lists the tasks (projects or tasks) for which you have the appropriate rights and the corresponding filter criteria apply.

Contact Prs. C.:

This field shows the contact person or the customer. By selecting a contact person, in some cases (e.g. project definition) a customer is automatically selected as well, since each contact person is assigned to exactly one customer.

A contact person is created via the input mask in the menu item "Master data".

1st Project L.:

In this field the manager of a project is displayed or must be selected from a selection list. The project manager is defined once in the master project, and is then preselected as a PROPOSAL in all underlying projects.

Green displayed persons in the selection field are project leaders, which are already assigned to at least one project in this role.

Project managers are created using the role mask with role "Project Manager".

Project Manag.:

Specifies which person should be the project manager of this project.

This field only serves as a further decision criterion in various reports, the Project Manager does not enjoy any additional rights or similar, these lie exclusively with the 1st Project Leader.

The Project Manager is only set in projects and is then also selected in the respective tasks of the project.

Green displayed persons in the selection field are project managers, which are already assigned to at least one project in this role.

Project managers are created using the role mask with role "Project Manager".

2nd Project L.:

Here you can specify a deputy project manager for a project, who will stand in for the project manager if the latter is absent.

Green displayed persons in the selection field are deputy project managers who are already assigned to at least one project in this role.

Deputy project managers are created using the role mask with role "project manager".

Date:

Used to enter or display a date of the day. If a date entry is required, the default value (today's date) is usually already preset.

Separator:

Separator for the export fields for project export in MS_Project.

Date Format:

Defines the format in which all dates are displayed and read in. The date must be defined with the following variables. Separator any:

dd: Day 2 digits
mm: Month 2 digits
yy: Year 2 digits
yyy: Year 4 digits

Date Bottom-Up:

Adjust date using the bottom-up method

Adjust the date according to the bottom-up method, i.e. all levels further up are stretched to the following date if there is a time overlap.

1509:

Text field

Variable text field that can be customized programmatically.

Insert Prefix:

Prefix (German: Präfix or also Vorsilbe genannt) refers to a string of characters preceding the root of a word, here to create a group of words that belong together.

It is used, for example, in the project name to achieve a uniform project affiliation already in the name. The placeholder structure to be used here can be defined in the administrator settings.

Field Descriptions

Row:

Specifies in which line of the mask this field should appear.

Project No.:

Sequential unique numbering.

Project:

This field displays the project name of the currently selected project.

Pln. Start:

The target start date of the project is displayed or specified here, i.e. from when the project is to be started according to planning.

Pln. End:

The target end date of the respective reference object, e.g. project, time data, trip data, etc. is displayed or specified here, i.e. by when the object should be completed according to planning.

Priority:

Indicates the urgency of the (sub)project.
The higher the priority of a project, the higher it is displayed in the control panel of an employee.

Plan. Eff.:

The target effort corresponds to the planned time effort in hours, man-days or man-years, etc. The estimated effort is entered when defining tasks. For projects/sub-projects, this value corresponds to the sum of all subordinate tasks.

Time Format:

With this user setting a user can define in which format he wants to have times displayed. The format is also the input format for this user.
hh:mm (hours, minutes each with two digits)
hh:mm:ss (hours, minutes, seconds each with two digits)
hh:mm
