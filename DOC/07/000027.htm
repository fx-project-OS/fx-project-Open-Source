Setting appointments incl. inviting people with automatic notification by e-mail


In this overview the comfortable input of own or foreign appointments is possible. The appointment can belong to the following categories:

  • Invitation
  • Business lunch
  • Conference
  • Contact
  • Meeting
  • Message
  • Private appointment
  • Miscellaneous
.
For each appointment, a responsible person must be specified, as well as the time period. Repeat appointments can also be created or edited here.

For each appointment type you can invite any person, who will be notified immediately by e-mail. Such invitations can be confirmed or rejected in the control center.




Selection

Select an appointment to view, change or delete.

Personal scheme:
Own appointments are always displayed at the top. The own person name is thereby bold deposited. Possible appointments of other persons are listed in alphabetical order, depending on the authorization.

Structure of an appointment line in the list:

  • First the period of the appointment is displayed.
  • If the appointment concerns you or if you have the right to change the appointment, a : colon followed by the appointment type is displayed. If a more detailed description is available for such appointments, it will also be displayed in brackets afterwards.
  • If the appointment is a recurrence appointment, {+} will be displayed at the end.
  • If the appointment is a reminder appointment, {!} will be displayed at the end.
  • If the appointment is already marked as "done" the whole line starts with an ! exclamation mark.
  • If the appointment cannot be changed, e.g. because of missing permission, the whole line is displayed enclosed in brackets. This mainly affects the display mode.
.

NOTE:
In display mode, appointments already marked as "done" are always displayed, otherwise only if a time restriction has been set in the time filter.

Field Descriptions

Default Action:

Normally "Display" should be entered here.

Person:

This selection field is used to select the person whose data is to be edited or displayed depending on the action.

App. Type:

The field "Appointment type" provides different appointment types for selection. While project managers can view all business appointments of an employee, you are not allowed to control private appointments.

Pln. Start Date:

The target start date of the project is displayed or specified here, i.e. from when the project is to be started according to planning.

Start of App.:

Specifies the start time for an appointment.

Repetition:

Specifies the recurrence of an appointment (daily, weekly, etc.).

End Repet.:

Specifies the day on which the repetition of an appointment ends.

Example: An appointment with repetition is created on 13.2. If, for example, "daily" is selected when creating the appointment and the end date of the repetition is 20.2., this appointment will be entered eight times between 13 and 20 February.

Reminder:

Date field for the reminder of an appointment.

If a date is entered, a reminder appointment for the current appointment will be created on this day.

End of App.:

Indicates the end of an appointment.

Duration:

Specifies whether an appointment extends over the whole day or only half a day. The respective duration depends on the definition of the working day, e.g. from 8 a.m. to 5 p.m. would correspond to a full-day appointment with a standard eight-hour day with one hour break.

If this field remains empty, the time information you entered is used.

No. Repet.:

number of repetitions

Here you can specify how often an appointment should be repeated, as an alternative to the end date.

Project:

The Project field lists the tasks (projects or tasks) for which you have the appropriate rights and the corresponding filter criteria apply.

Time-Rec. App.:

A time recording appointment is an appointment that is listed in the control center and can be transferred to the time recording with a simple click.

If a time recording appointment is assigned to a project/subproject, then the time recording is booked to the next following appointment project. On the other hand, if the appointment is not assigned to a project, then it is posted to [Unknown].
Application examples are invitation appointments to project meetings, business meals etc.

Meeting Loc.:

This field indicates the meeting point of an appointment (should only be used as an aid).

Finished:

If this box is checked, it does not mean that the appointment is deleted, it is just marked as "done".

Reference:

Specifies the subject of an appointment.

Description:

Explanatory descriptions, details, comments or similar.


Save

By clicking on this button, the changes are made. The text of the button depends on the current action mode.

Checking

check inputs

By clicking on this button, the entries are checked for correctness or completeness without immediately writing the entries to the database.
