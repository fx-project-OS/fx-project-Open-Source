Administration > Profile/User > User Accounts

Activate, deactivate or generate a new password for users


User accounts (user accesses) can be deactivated if required, i.e. the associated user profiles with the assigned license points are credited to the points account again. The same applies to activation, but in the opposite way. In addition, locked users whose access was locked after multiple incorrect password entries can be reactivated. In addition, a new password can be generated at the request of a user, which will be sent to the respective user by e-mail.




Status control

Action selection

From Status control, Generate Password, Lift Lock, Activate User, and Deactivate User get to User status.



Field Descriptions


Display

Generate Passw.

In Generate Password a new password is generated and sent to the user via email.

Lift Lock

In Lift Lock, users who have repeatedly entered their password incorrectly are unlocked.

Activate User

Enable user

In Activate User inactive users are reactivated.

Deactivate User

In Deactivate User active users are set to inactive.

Date:

Used to enter or display a date of the day. If a date entry is required, the default value (today's date) is usually already preset.

Status:

The current status of a user is stored in this field:
0 - active (without SYS password) = without password automatically generated by the system after lock etc.
1 - active (with SYS password) = with system-generated password via [Password gen.].
2 - inactive
3 - locked

By clicking on this button, the changes are made. The text of the button depends on the current action mode.
