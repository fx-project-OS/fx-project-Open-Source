Listing and evaluation of the selected projects on the basis of the schedule deviation and the degree of completion based on target/actual comparison


Listing and evaluation of the selected projects on the basis of the schedule variance as well as the degree of completion based on the target/actual comparison.

If selected, the report can also display history information up to 12 months in the past or the selected figures in a graphical chart.




Select project

Select the start operation for which the performance evaluation is to be created.



Field Descriptions

Project:

The Project field lists the tasks (projects or tasks) for which you have the appropriate rights and the corresponding filter criteria apply.

History:

If this checkbox is activated, any existing historical data records for each project are also displayed. These old data records are displayed with a blue project number and project name.

Detailed:

By ticking this checkbox, the normal report is extended by additional, more detailed information.

Picture:

Assigned image in PNG, JPG, GIF or BMP format.


Display

By clicking on this button, the list is generated and displayed. Depending on the defined selection criteria, the list is filtered accordingly.




Project list

Listing of key figures per project incl. evaluation of past performance.



Field Descriptions

Project:

This field displays employees who are involved in the current project.

Stat.:

This field shows how far a project has already progressed. In input mode, you can assign an attribute from a selection list to a project here, which reflects whether or how far a project is already advanced.

The status can also be used to decide whether project data should be included in a report or not.

Start:

The target start date of the project is displayed or specified here, i.e. from when the project is to be started according to planning.

End:

The target end date of the respective reference object, e.g. project, time data, trip data, etc. is displayed or specified here, i.e. by when the object should be completed according to planning.

Days:

Display of the days after the target start date

Plan.:

Display of target hours

Text:

Display of the unit: h = hours

Act.:

Display of actual hours

Prod.:

Display of productive hours (e.g. production, support, etc.)

I-Hrs:

Display of billable hours

PAC:

Shows the percentage of a project that has already been completed. The planning effort is put in relation to the hours worked.

Any delays are taken into account in the detailed calculation.

Example:

  • without delays:
    For a task with a target effort of 100 hours, 70 hours of actual effort have already been performed => the project is 70% (=> 70 / 100) completed.
  • With delays:
    For a task with a target effort of 100 hours, 70 hours of actual effort have already been done, but there was a 5 hour delay => the project is 66% (=> 70 / 105) completed.
