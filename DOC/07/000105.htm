Listing of all ougoing AR invoices with the option of entering incoming payments


Manage (view or modify) invoice lists selected by "Client", "Project" or "Period".




Search criteria

Entering search criteria for invoices

Here you can enter search criteria for invoices. According to these criteria lists of existing invoices are displayed.

Button "Invoices"- displays the existing invoices for selection.

Button "Outgoing list"- Outgoing information about selected invoices.

At action "Create record [2]- you can enter criteria for invoice items, e.g. for which customer and for which project an invoice is created, as well as the time span in which the trips or time entries can be searched.
With action "Change record [3]-these values are taken over from the changed invoice.

Checkbox "with travel costs", "with material", "project with fixed price", "project with hourly basis: corresponding costs are displayed.

Checkbox "Unknown": also entries of unknown projects will be displayed.



Field Descriptions

Customer:

Here you can select the sold-to party of the task (project, subproject or task).

Time Span from:

This field displays the point in time from which data for a project should be listed.

In input mode, a date can be entered here according to the user settings.

Search from:

In this search field you can define the validity start of the time frame of the search.

Text:

Indicator in the form of a traffic light function whether invoice was already paid.

Time Span to:

This field displays the date up to which the data for a project should be listed.

In input mode, a date can be entered here according to the user settings.

Search up to:

In this search field you can set the validity end of the time frame of the search.

Project:

The Project field lists the tasks (projects or tasks) for which you have the appropriate rights and the corresponding filter criteria apply.

Column:

Specifies in which column the field should appear in the mask.

Invoice Type:

Type of invoice, which is the heading above the PDF-generated invoice.

The invoice types can be stored in the [communication data] of the persons, e.g. in the client basic data, so that these invoice types apply to the entire client.

The default invoice type is [Invoice] as the heading for an invoice.

Ignore Release:

Ignore release

Calculat. Type:

The calculation type determines whether deduction items from previous (partial) invoices should be displayed (type = complete) or not (type = normal).

Invoice Form:

Select which of the existing invoice forms of the client folders should be used for the PDF preparation of the invoice.

Invoiced Positions:

If this checkbox is set, all positions that have already been invoiced will also be displayed.

Grouping Pos.:

Maximum number of bundle items to be created per invoice category.

A bundle item can consist of any number of invoice items.
The individual invoice items to be bundled must have the same VAT rate!
The quantity for the bundled item is always 1.00 and the unit is [Div.] (for miscellaneous), otherwise it will cause problems if different units like [Std.] and [kg] etc. are to be bundled.
The net price of the bundled item is equal to the sum of the net prices times the quantity of the individual assigned invoice items.

Invoice created:

Indicates whether an invoice has already been created or not.

Invoice paid:

Indicates whether an invoice has already been paid or not.

Cancel Invoice:

"Yes" - invoice is marked as "cancelled" (instead of deleted), invoice positions in "Time data" are released (new invoice can be taken over again).

Invoice list

Invoice list

Display invoices according to search criteria.



Field Descriptions

Status:

Displays the status of a record.

Invoice Number:

The placeholder for the invoice number is an admin setting value.

Date:

Used to enter or display a date of the day. If a date entry is required, the default value (today's date) is usually already preset.

Invoice Type:

Type of invoice, which is the heading above the PDF-generated invoice.

The invoice types can be stored in the [communication data] of the persons, e.g. in the client basic data, so that these invoice types apply to the entire client.

The default invoice type is [Invoice] as the heading for an invoice.

Calc. Type:

The calculation type determines whether deduction items from previous (partial) invoices should be displayed (type = complete) or not (type = normal).

Person:

This selection field is used to select the person whose data is to be edited or displayed depending on the action.

Contact:

This field shows the contact person or the customer. By selecting a contact person, in some cases (e.g. project definition) a customer is automatically selected as well, since each contact person is assigned to exactly one customer.

A contact person is created via the input mask in the menu item "Master data".

City/Town:

The person's place of residence is entered in this field.

Project:

The Project field lists the tasks (projects or tasks) for which you have the appropriate rights and the corresponding filter criteria apply.

Net Amount:

Invoice costs without VAT.

Value-Add. Tax:

VAT for invoice costs.

VAT-Rate:

Statutory value-added tax rate at which the invoice is issued. The gross amount to be billed is calculated from this tax rate.

Gross Amount:

Invoice costs including statutory value added tax

Sent on:

Indicates when the invoice went out.

Due Date:

The key date is the day of the expected receipt of payment, that is, the day by which an outgoing invoice should be paid at the latest.

If no cut-off date is defined, the day 14 days after the invoice issue date is used by default.

Paid on:

Indicates when the invoice was paid.

Amount Paid:

How much of the bill has been paid?
