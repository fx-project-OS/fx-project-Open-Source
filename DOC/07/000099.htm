Creation > Projects > Copy

Copy a task, a subproject or a complete main project to another project


Enables the copying of entire project hierarchies incl. subprojects and optional resource allocation from a source process to any target level. In doing so, the deadline period can be conveniently moved in the target frame, the project main responsible persons can be adjusted and the project titles can be individually changed.

By copying, a new parent project can easily be created from a subproject.


NOTE:
It should be noted here that when copying subprojects, the deadline period must fit in the period of the target project.




Copy projects

Copy projects

Select here which project is to be copied and, if necessary, assign an item to it ("Assigned to"). You must also specify when the new project is to start.

The ticked checkbox Displ. Sub Projects causes all subprojects of the selected project to be copied as well, just as ticking the field "Take over resources" takes over all resources of the template.

The reference numbers of copied projects are provided with a "(c*)" character.

Example: original Ref-No.: A6ZhdR copy => Ref-No.: A6ZhdR (c*)



Field Descriptions

Selection of the activity (project, subproject, task) for which you are authoriz:

In Project the task (project or task) to be copied is selected.

Higher-level unit:

This field contains the "parent unit".
For projects, this would be the parent project [(sub)project].

For documents this would be the unit you belong to (e.g. project, appointment, note etc.).

Displ. Sub Projects:

If the checkbox is activated, all subordinate subprojects and tasks are included in addition to the main project.

Attention!
This can lead to longer waiting times for large projects.

Copy Resources:

If this field is checked, the resources that are assigned to a task are also copied.

Transfer Dest. Value:

If this checkbox is ticked, the additional information is pre-populated with the values of the selected target project, i.e. project manager, project leader, etc.


Select

Additional information

Additional information for the copy routine.

Details of additional information to be taken into account during the copying process, such as client, project manager, project manager, etc.

NOTE:
A stretching or compression of the project deadlines is not yet supported, but a deadline adjustment is supported, if no deadlines are affected.

Field Descriptions

Contact Prs. C.:

Here the contact person of the ordering party is selected.

Project Manager:

Here the project manager of the task (project, subproject or task) is selected.

1st Project L.:

Here the project manager of the activity (project, subproject or task) is selected.

2nd Project L.:

The deputy project manager of the activity (project, subproject or task) is selected here.

Pln. Start Date:

The target start date of the project is displayed or specified here, i.e. from when the project is to be started according to planning.

Project Categories:

Define filter categories for projects or select them in a selection field.

Pln. End Date:

The target end date of the respective reference object, e.g. project, time data, trip data, etc. is displayed or specified here, i.e. by when the object should be completed according to planning.

Determinat. Method:

Determination of the method how the project progress (= percentage of completion) is to be determined.

Report Date:

Determination of the reporting date when the project progress (= percentage of completion) is to be determined.


Accept
