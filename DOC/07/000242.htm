Control > Cost Accounting > Cost Main Data: Relations

Linking of activity types to cost types


Establish 1:n relationships between a cost element and any number of activity types.




Select cost element for relationships with activity types

Select a cost element to which you want to assign individual activity types or change existing assignments.



Field Descriptions

Cost Center:

Here cost type: Possible assignment type of the individual cost master definitions, such as project, person, material, etc.

Search Text:

This field is used to enter the string to search for.

Cost Number:

Unique identification number of cost master data such as cost objects, cost centers, cost elements or activity types.

Cost Label:

Designation of a unique cost center.

Cost Center Type:

Possible assignment type of the individual cost master definitions, such as project, person, material, etc.

Validity from:

Date from which a record is valid.

Validity to:

Date until when a record is valid.


Display

By clicking on this button, the list is generated and displayed. Depending on the defined selection criteria, the list is filtered accordingly.




Assign activity type(s) to a cost element

By simply selecting the checkboxes, a wide variety of activity types can be assigned to a cost element. The displayed activity types are determined on the basis of the selected cost type, i.e. the validity area and cost center type must match.

NOTE:
The assignment of activity types for which time recording data has already been made can no longer be changed.



Field Descriptions

Only Cust. Data:

Activate or deactivate an entry by placing a tick in the "checkbox".

Service Types:

The activity type in cost accounting is a reference value that indicates the qualitative technical description of the production activity. Here, it can be classified between productive (=value-adding) and non-productive types.

Prod.:

Determines whether an activity type is productive (= value-added) or not.

T-Chk:

To control the daily time recording, a check program can run in the scheduler which checks whether the time recording has been carried out correctly.

If this value is set, the check is carried out in the scheduler and the employee receives a notification if there is no or an incorrect time recording.

If this value is not set, the check in the scheduler is not performed.

Valid from:

Date from which a record is valid.

Valid to:

Date until when a record is valid.


Save

By clicking on this button, the changes are made. The text of the button depends on the current action mode.
